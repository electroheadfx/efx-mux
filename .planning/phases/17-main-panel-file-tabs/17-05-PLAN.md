**file-service.ts -- Trigger git refresh after save (UAT Gap 2a):**

The Rust file watcher watches `.git/` directory for index changes and emits `git-status-changed`. But saving a file via `writeFile` modifies the working tree, not .git/index. The sidebar's git-status-changed listener only fires when .git/index changes (e.g., after `git add`). So the sidebar never refreshes after a plain file save.

Fix: After `writeFile` completes successfully, dispatch a custom DOM event that the sidebar and git-changes-tab can listen for. Use the existing `git-status-changed` pattern but via a DOM CustomEvent (since the Tauri event system requires the backend):

```typescript
export async function writeFile(path: string, content: string): Promise<void> {
  try {
    await invoke('write_file_content', { path, content });
    // Notify frontend components that a file was written (triggers git status refresh)
    document.dispatchEvent(new CustomEvent('file-written', { detail: { path } }));
  } catch (e) {
    throw new FileError('WriteError', String(e));
  }
}
```

Then in sidebar.tsx (already imported via listener) -- but actually, the sidebar already listens for `git-status-changed` Tauri events. The simplest approach: make the sidebar ALSO listen for `file-written` DOM events by adding a listener in the existing useEffect. BUT since the sidebar is NOT in this plan's files_modified, instead trigger the refresh differently:

Alternative approach (simpler, no sidebar changes needed): After writeFile succeeds, invoke the Rust backend to re-check git status and emit the event:

```typescript
export async function writeFile(path: string, content: string): Promise<void> {

    // but saving a working tree file changes the diff vs HEAD.

    throw new FileError('WriteError', String(e));

```

The sidebar needs to listen for this event. Since sidebar.tsx is in this plan's files_modified list... actually it's in Plan 04. Let me adjust the approach.

Best approach: In `editor-tab.tsx`, after a successful save, call `invoke('get_git_status', ...)` to trigger a re-fetch. But that doesn't notify the sidebar.

Simplest correct approach: Dispatch a custom `file-written` event from `writeFile`. Then in `editor-tab.tsx` (which calls writeFile), after the save succeeds, also dispatch a Tauri-compatible notification. The sidebar already listens for `git-status-changed` from Tauri. We can use `emit` from `@tauri-apps/api/event` to emit a frontend-only event with the same name:

```typescript
import { emit } from '@tauri-apps/api/event';

export async function writeFile(path: string, content: string): Promise<void> {
  try {
    await invoke('write_file_content', { path, content });
    // Trigger git status refresh for sidebar and git-changes-tab.
    // The file watcher only watches .git/index, not working tree changes.
    // Emit the same event name so existing listeners pick it up.

    throw new FileError('WriteError', String(e));
  }

```

The `emit` from `@tauri-apps/api/event` emits to frontend listeners (from `listen()`). This is exactly what the sidebar and git-changes-tab use. This is the cleanest fix -- no new event names, no new listeners needed.

**editor-tab.tsx -- Refresh content after external file change (UAT Gap 2b):**

When a file is reverted via `git checkout` in the terminal, the editor tab still shows the old content. The editor needs to detect external file changes and reload.

The Rust file watcher already emits `md-file-changed` for .md files but not for all files. Rather than modifying the Rust watcher (out of scope), use a pragmatic approach:

1. Listen for the `git-status-changed` event (which fires after git operations like checkout/revert). When it fires, re-read the file from disk and compare with the editor's saved content. If different, update the editor.

2. Add a `useEffect` in EditorTab that listens for `git-status-changed`:

```typescript
// Re-read file content when git status changes (handles git checkout/revert)
useEffect(() => {
  let cancelled = false;
  async function handleGitChange() {
    if (cancelled || !viewRef.current || !setupRef.current) return;
    try {
      const diskContent = await readFile(filePath);
      const savedContent = setupRef.current.getSavedContent();
      if (diskContent !== savedContent) {
        // File changed on disk -- update editor
        const view = viewRef.current;
        view.dispatch({
          changes: { from: 0, to: view.state.doc.length, insert: diskContent },
        });
        setupRef.current.setSavedContent(diskContent);
        setEditorDirty(tabId, false);
      }
    } catch {
      // File may have been deleted -- ignore
    }
  }
  const unlisten = listen('git-status-changed', handleGitChange);
  return () => {
    cancelled = true;
    unlisten.then(fn => fn());
  };
}, [filePath, tabId]);
```

This requires importing `listen` from `@tauri-apps/api/event` and `readFile` from `../services/file-service` in editor-tab.tsx. Also import `setEditorDirty` (already imported).

Add the import for `listen`:
```typescript

```

Add the import for `readFile`:
```typescript

```

**terminal-tabs.tsx -- Fix tab type persistence (UAT Gap 4):**

The root cause: Tab persistence saves `{ sessionName, label }` but NOT whether the tab is an agent tab or a plain Zsh tab. On restore (line 624), the logic `(i === 0 && agentBinary) ? agentBinary : undefined` always gives the agent binary to the first tab -- but the first tab may have been a plain Zsh tab (user may have closed the agent tab and the first remaining tab is Zsh).

Additionally, "Zsh tabs restored as agent after restart" happens because `initFirstTab` (line 296-297) assigns the agent label ("Claude"/"OpenCode") to the first tab unconditionally if the project has an agent configured.

Fix:

1. **Add `isAgent` field to persisted tab data.** Change `persistTabState` to include whether each tab is an agent tab:

```typescript
function persistTabState(): void {
  const activeName = activeProjectName.value;
  const tabs = terminalTabs.value.map(t => ({
    sessionName: t.sessionName,
    label: t.label,
    isAgent: t.isAgent ?? false,
  }));
  const data = JSON.stringify({ tabs, activeTabId: activeTabId.value });
  const patch: Record<string, string> = { 'terminal-tabs': data };
  if (activeName) {
    patch[`terminal-tabs:${activeName}`] = data;
  }
  updateSession(patch);
}
```
