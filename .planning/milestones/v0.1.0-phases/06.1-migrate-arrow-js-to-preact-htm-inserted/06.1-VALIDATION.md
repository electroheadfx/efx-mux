## Nyquist Rationale

This phase is a big-bang framework migration where all components are rewritten. The primary automated verification is `npm run build` which runs `tsc --noEmit && vite build` -- this catches type errors, missing imports, invalid JSX, and broken module resolution across ALL files in every wave. Combined with per-task grep checks for structural correctness (e.g., `terminal-area` class presence, no `@arrow-js/core` remnants), this provides continuous automated feedback. vitest.config.ts is created in Wave 0 for future unit tests but the build-based verification is the primary quality gate for a migration phase.
