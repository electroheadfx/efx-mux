## Component Inventory

### 1. Terminal Tab Bar (main panel, top edge)

**Location:** Top of `.terminal-area` in main-panel.tsx, above all terminal containers.

**Visual spec:**
- Container: `flex gap-1 px-2 py-1.5 bg-bg-raised border-b border-border shrink-0 items-center h-[32px]`
- Matches existing TabBar from tab-bar.tsx but with close button per tab
- Active tab pill: `px-3 py-1 text-xs bg-accent text-white rounded-full border border-accent`
- Inactive tab pill: `px-3 py-1 text-xs bg-transparent text-text rounded-full border border-transparent hover:text-text-bright hover:bg-bg-raised/40`
- Close button per tab: `ml-1 text-[10px] text-text opacity-60 hover:opacity-100 hover:text-[#dc322f]` showing unicode multiply sign
- New tab button at far right: `w-6 h-6 text-text text-sm rounded hover:bg-bg hover:text-text-bright` showing "+"

**Interaction:**
- Click tab to switch (show/hide terminal containers via `display:none/block`)
- Click close button on tab to close that tab (D-06)
- Click "+" to create new tab (same as Ctrl+T)
- Ctrl+Tab cycles left to right, wraps around
- After tab switch: call `terminal.focus()` + `fitAddon.fit()` on the newly active terminal

**Naming convention:** Tabs labeled "Terminal 1", "Terminal 2", etc. Auto-increment counter. If project agent is "claude", first tab label is "Claude" instead of "Terminal 1".

### 2. Shortcut Cheatsheet Overlay (Ctrl+?)

**Location:** Fixed overlay, centered in viewport.

**Visual spec:**
- Backdrop: `fixed inset-0 bg-black/30 z-[100]` (matches fuzzy-search.tsx pattern)
- Card: `w-[420px] max-h-[70vh] bg-bg-raised border border-border rounded-lg shadow-[0_8px_32px_rgba(0,0,0,0.4)] overflow-y-auto`
- Header: `px-6 py-4 border-b border-border text-text-bright text-base` -- copy: "Keyboard Shortcuts"
- Each row: `flex items-center justify-between px-6 py-2 border-b border-border/50`
- Key badge: `inline-flex items-center px-2 py-0.5 bg-accent/15 text-accent text-xs font-mono rounded` -- e.g. "Ctrl+T"
- Action label: `text-sm text-text-bright` -- e.g. "New terminal tab"
- Section divider: `px-6 py-2 text-[11px] uppercase tracking-widest text-text` -- e.g. "Terminal", "Navigation", "App"

**Shortcut table content:**

| Section | Key | Action |
|---------|-----|--------|
| Terminal | Ctrl+T | New terminal tab |
| Terminal | Ctrl+W | Close active tab |
| Terminal | Ctrl+Tab | Next tab |
| Navigation | Ctrl+P | Switch project |
| Navigation | Ctrl+B | Toggle sidebar |
| Navigation | Ctrl+S | Toggle server pane |
| App | Ctrl+Shift+T | Toggle dark/light theme |
| App | Ctrl+? | This cheatsheet |

**Interaction:**
- Opens on Ctrl+? (Ctrl+Shift+/ on US layout, also bind Ctrl+/ as AZERTY fallback)
- Dismisses on: Escape key, any click outside card, any other shortcut key press
- No focus trap needed (informational only, not a form)

### 3. PTY Crash/Exit Overlay (per terminal tab)

**Location:** Absolute positioned inside the terminal tab container (not viewport-fixed). Only visible for the specific tab whose PTY exited.

**Visual spec -- normal exit (code 0):**
- Backdrop: `absolute inset-0 bg-black/50 flex items-center justify-center z-20`
- Card: `bg-bg-raised border border-border rounded-lg p-6 text-center max-w-[320px]`
- Status dot: `w-2.5 h-2.5 rounded-full bg-[#859900] mx-auto mb-3`
- Message: `text-text-bright text-sm mb-4` -- copy: "Session ended"
- Restart button: `bg-accent text-white px-4 py-2 rounded-sm text-sm cursor-pointer hover:opacity-90 transition-opacity` -- copy: "Restart Session"

**Visual spec -- crash (non-zero exit code):**
- Same layout as normal exit
- Status dot: `bg-[#dc322f]` instead of green
- Message: `text-text-bright text-sm mb-1` -- copy: "Process crashed"
- Exit code detail: `text-text text-xs mb-4` -- copy: "Exit code {N}"
- Restart button: same as normal exit

**Interaction:**
- Appears when `pty-exited` Tauri event fires for this tab's session
- Terminal content remains visible behind the dimmed backdrop (scrollback preserved)
- Click Restart: dismiss overlay, spawn new PTY with same session type (agent or bash)
- The overlay is per-tab -- other tabs are unaffected

### 4. First-Run Wizard Modal

**Location:** Fixed modal, same pattern as project-modal.tsx. Replaces the existing `openProjectModal({ firstRun: true })` code path.

**Trigger:** On app launch when `state.json` is missing or contains zero projects.

**Visual spec:**
- Backdrop: `fixed inset-0 bg-black/50 z-[100] flex items-center justify-center`
- Card: `w-[520px] bg-bg-raised border border-border rounded-lg shadow-2xl`
- Header: `px-6 py-4 border-b border-border flex items-center justify-between`
- Step title in header: `text-base text-text-bright`
- Step indicator: row of 5 dots below header, `flex gap-2 justify-center py-3`
  - Active dot: `w-2 h-2 rounded-full bg-accent`
  - Inactive dot: `w-2 h-2 rounded-full bg-border`
  - Completed dot: `w-2 h-2 rounded-full bg-accent opacity-50`
- Body: `px-6 py-4` (each step renders different fields)
- Footer: `px-6 py-4 border-t border-border flex justify-between items-center`
  - Skip link (left): `text-sm text-text cursor-pointer hover:text-text-bright` -- copy: "Skip"
  - Back button (left of primary, when not step 0): `bg-transparent border border-border text-text px-4 py-2 rounded-sm text-sm cursor-pointer hover:bg-bg hover:text-text-bright`
  - Primary CTA (right): `bg-accent text-white px-4 py-2 rounded-sm text-sm cursor-pointer hover:opacity-90` -- copy changes per step (see below)

**Wizard steps:**

**Step 0 -- Welcome**
- Heading: "Welcome to Efxmux"
- Body copy: `text-sm text-text leading-relaxed` -- "A terminal workspace for AI-assisted development. Let's set up your first project."
- Primary CTA: "Get Started"
- No Skip link on this step

**Step 1 -- Project Setup**
- Heading: "Add Project"
- Fields: Directory (with Browse button) + Name (auto-filled from dir basename)
- Reuse exact field markup from project-modal.tsx (directory input + browse button, name input)
- Auto-detect .planning/ dir for GSD file (existing handleBrowse logic)
- Primary CTA: "Next" (enabled when directory + name both non-empty)
- Skip: sets directory to home dir, name to "default"

**Step 2 -- Choose Agent**
- Heading: "Choose Agent"
- Body copy: "Which AI coding agent do you use?"
- Three option cards in a row: `flex gap-3`
  - Each card: `flex-1 p-4 rounded border cursor-pointer text-center transition-all duration-150`
  - Selected: `border-accent bg-accent/10 text-text-bright`
  - Unselected: `border-border bg-bg text-text hover:border-accent/50`
  - Card content: agent name in 14px semibold + description in 11px muted text
  - Claude: "Claude Code" / "Anthropic's AI agent"
  - OpenCode: "OpenCode" / "Open-source alternative"
  - Bash: "Plain Shell" / "No AI agent"
- Primary CTA: "Next"
- Skip: defaults to "bash"

**Step 3 -- Theme Import**
- Heading: "Import Theme"
- Body copy: "Import your iTerm2 color profile for a familiar look."
- Import button: `bg-bg border border-border text-text-bright px-4 py-2 rounded-sm text-sm cursor-pointer hover:border-accent` -- copy: "Choose iTerm2 Profile..."
- After import: show preview swatch row (6 color circles from imported theme)
- Primary CTA: "Next"
- Skip: keeps default Solarized Dark theme

**Step 4 -- Server + GSD**
- Heading: "Dev Server & GSD"
- Two fields (reuse project-modal.tsx field markup):
  - Server Command: placeholder "Optional, e.g. npm run dev"
  - GSD File: placeholder "Optional .md path" (pre-filled if auto-detected in Step 1)
- Primary CTA: "Finish Setup"
- Skip: no server, no GSD file

**After final step:** Call `addProject()` with collected data, close wizard, app loads normally.

**Interaction:**
- Escape does NOT close wizard (same as current first-run modal behavior)
- Close button (X) in header always available -- closes wizard with defaults for all unanswered steps
- Back button navigates to previous step
- Skip advances to next step without saving current step's data
- Keyboard: Enter triggers primary CTA, Tab moves between form fields

## Accessibility Notes

- Tab bar uses `role="tablist"` with `role="tab"` on each tab pill and `aria-selected` on active tab
- Crash overlay uses `role="alertdialog"` with `aria-labelledby` pointing to the message text
- Wizard modal uses `role="dialog"` with `aria-modal="true"` and focus trap (Tab cycles within modal)
- Cheatsheet overlay uses `role="dialog"` (informational, no focus trap needed)
- All buttons have explicit `title` attributes for tooltip text
- Shortcut cheatsheet is informational -- screen readers can navigate the shortcut table
