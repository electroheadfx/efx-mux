## Spacing Scale

> **Project-level spacing exception acknowledgment:** The compressed spacing scale from `src/tokens.ts` is a locked project constraint that predates Phase 22. Values such as 1px, 2px, 6px, and 10px exist as established project tokens, not as new design debt introduced by this phase. The 18px button sizes (split icon, preferences button) match the existing `.titlebar-add-btn` w/h = 18px pattern already shipped in prior phases. These token values are project token references, not newly declared spacing values for Phase 22.

This project uses a **compressed spacing scale** (not the standard 8-point grid). All values are from `src/tokens.ts` `spacing` object and are the canonical source of truth. Do not use arbitrary values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 1px | Micro gaps (e.g. icon sub-pixel offsets) |
| sm | 2px | Tight inline gaps |
| md | 4px | Default element padding (tab bar icon padding) |
| lg | 6px | Compact row padding |
| xl | 8px | Standard button / tab padding |
| 2xl | 10px | Larger tab content padding |
| 3xl | 12px | Section padding |
| 4xl | 16px | Layout-level spacing |
| 5xl | 20px | Major element spacing |
| 6xl | 28px | Section breaks |

Source: `src/tokens.ts` — locked; do not add new spacing values.

Exceptions for Phase 22:
- Split handle hit target: 8px (matches existing `--handle-hit: 8px` CSS variable)
- Split icon button in tab bar: 18x18px (matches `.titlebar-add-btn` w/h = 18px — already shipped)
- Preferences button in titlebar: 18x18px (same as split icon — mirror of `.titlebar-add-btn` — already shipped)
- Intra-zone resize handle rendered height: 1px visual / 8px hit area (matches `.split-handle-h` pattern — already shipped)

Source: `src/tokens.ts` (verified by read) + `src/styles/app.css` @theme (verified by read).

**Primary focal point:** The active tab's bottom underline (`#258AD1`, `var(--color-accent)`) in the topmost scope's `UnifiedTabBar`. This is the single most visually prominent accent element in the idle workspace.

**Accent (`#258AD1`) is reserved for:**
1. Split handle (`.split-handle-h` and new `.split-handle-h-intra`) on hover and while dragging
2. Zone-level vertical split handles (`.split-handle-v`) on hover and dragging — already established
3. Drop affordance border on target tab bar during cross-scope drag (`.drop-target` CSS class — new, accent outer border)
4. Active tab bottom underline (existing pattern in UnifiedTabBar)
5. Preferences button hover state (`.titlebar-prefs-btn:hover` — mirrors `.titlebar-add-btn:hover`)
6. Split icon button hover state (`.tab-bar-split-icon:hover` — same hover style as `+` menu trigger)
7. Focus outline on interactive elements (existing: `file-tree:focus` pattern)

Accent is NOT used for: inactive tab labels, tab bar background, split handle idle state, empty scope background, disabled menu item backgrounds.

## Component Inventory

New CSS classes required in `src/styles/app.css` (sourced from CONTEXT.md D-15, D-08, D-09, and RESEARCH.md Pattern 4):

Mirror of `.titlebar-add-btn` (app.css:444). Position: right side of `.titlebar-drag-region`.

```
width: 18px
height: 18px
border-radius: 4px (radii.md)
background: transparent

-webkit-app-region: no-drag

```

Hover: `color: var(--color-accent); border-color: var(--color-accent);`
Active: same hover style (no orange active — differs from `.titlebar-add-btn:active` which is `#e67e22`)

Icon: Lucide `Settings`, 18px (D-16). Source: RESEARCH.md icon selection table.

### `.tab-bar-split-icon`

New button in every `UnifiedTabBar`, positioned immediately after the `+` menu trigger (D-08).

```
width: 18px
height: 18px
border-radius: 3px (radii.sm)
background: transparent
color: var(--color-text)           /* textMuted — 8B949E */
cursor: pointer
display: flex; align-items: center; justify-content: center;
transition: color 0.15s, background-color 0.15s;
```

Hover: `color: var(--color-accent); background: var(--color-accent)10;` (subtle tint)
Disabled (cap of 3 reached): `color: var(--color-text-muted); opacity: 0.4; cursor: not-allowed;`

When disabled: set `aria-disabled="true"` and `aria-label="Split pane (maximum 3 panes reached)"` on the button element.

Icon: Lucide `Rows2`, 14px (RESEARCH.md recommendation — `SplitSquareVertical` does not exist in lucide-preact 1.8.0).

### `.split-handle-h-intra`

Intra-zone horizontal resize handle between stacked sub-panes (D-09). Reuses exact `.split-handle-h` pattern.

```
width: 100%
height: 1px (--handle-size)
min-height: 1px
cursor: ns-resize        /* ← ns-resize for top/bottom, not row-resize */
background: var(--color-border)
background-clip: content-box
flex-shrink: 0
margin: calc((8px - 1px) / -2) 0
padding: calc((8px - 1px) / 2) 0
z-index: 10
transition: background 0.1s ease
```

Hover + `.dragging`: `background: var(--color-accent); background-clip: content-box;`

Note: `cursor: ns-resize` (CONTEXT.md D-09) is the semantic cursor for top/bottom resize. The existing `.split-handle-h` uses `row-resize` — Phase 22 intra-zone handles MUST use `ns-resize` to be visually distinct.

### `.drop-target`

Applied to the `[data-tablist-scope]` wrapper of the target tab bar during cross-scope drag (D-14, RESEARCH.md Pattern 4).

```
outline: 1px solid var(--color-accent)
outline-offset: -1px
```

Applied via class toggle in `onTabMouseMove` when cursor crosses scope boundary. Cleared in `cleanupReorder`. No pane body tint (D-14 explicit).

### `.scope-empty-placeholder`

Shown in a scope body when all tabs are closed (D-03).

```
display: flex
align-items: center
justify-content: center
height: 100%
color: var(--color-text-muted)   /* textDim — 556A85 */
font-size: 11px (fontSizes.base)
font-family: var(--font-family-sans)
font-weight: 400
user-select: none
```

## Interaction Contracts

### Tab Close (uniform dynamic behavior — D-01)

- All tab types show `×` close button on hover (including File Tree, GSD, Git Changes).
- Click `×` on File Tree / GSD / Git Changes: closes tab. No confirmation required (no unsaved state in these tab types).
- Click `×` on editor tab with unsaved changes: existing confirm-modal flow (Phase 17). Unchanged.
- Click `×` on agent/terminal tab: existing agent-quit-confirm flow (quick-260416-h3i). Unchanged.
- Closing last tab in a scope: scope body shows `.scope-empty-placeholder` (D-03). No last-tab guard.

### Singleton Dimming (D-04)

- `+` menu item for GSD: disabled (opacity 0.4, pointer-events none) in every scope where GSD is not the current `owningScope` AND GSD is already open somewhere.
- `+` menu item for Git Changes: same dimming rule.
- Dimmed item tooltip / caption: "Already open in another pane" — shown on hover (11px, `textDim`).
- File Tree: NOT a singleton. `+` menu item always enabled. Multiple scopes can have File Tree tabs simultaneously.

### Split Icon (D-07 / D-08)

- Click split icon when zone has fewer than 3 sub-panes: spawns a new empty sub-pane below the current scope in that zone. Transition: instant (no animation — matches project's zero-animation approach for panel spawns).
- Click split icon when zone is at cap (3 sub-panes): icon is disabled (see `.tab-bar-split-icon` disabled state above). Disabled state uses `aria-disabled="true"` and `aria-label="Split pane (maximum 3 panes reached)"`. No visual tooltip required — the cap of 3 is visually obvious.
- Split icon default aria-label (enabled state): "Split pane".
- Split icon position: immediately right of the `+` menu trigger in the tab bar. Left of any right-aligned tab bar actions.

### Intra-Zone Resize Handle (D-09)

- Drag `.split-handle-h-intra` between sub-panes: resizes the two adjacent sub-panes proportionally.
- Minimum sub-pane height: 48px (prevents collapse below usable size).
- Cursor: `ns-resize` during handle hover and drag.
- Ratio persisted in `AppState.layout` as `main-ratios: number[]` / `right-ratios: number[]` (parallel to existing `sidebar-main` / `main-right` ratio pattern).

### Cross-Scope Drag (D-11 / D-13 / D-14)

- Drag source: any tab in any scope (Terminal, Agent, Editor, GSD, Git Changes, File Tree).
- Mouse-based drag (`mousedown` → `mousemove` → `mouseup`) — HTML5 DnD is off (WKWebView hijacks it).
- During drag: ghost follows cursor (existing behavior). When cursor enters a different scope's tab bar, that tab bar gains `.drop-target` outline.
- Drop on target: tab is moved to target scope. `ownerScope` is updated.
- GSD / Git Changes singleton move: `owningScope` flipped to target. Source scope `+` menu re-enables the singleton item. Target scope `+` menu dims it.
- PTY session name: unchanged on drag (D-12). Only `ownerScope` metadata changes.

### Preferences Button (D-15 / D-17)

- Button rendered in `.titlebar-drag-region`, right side (after the drag region ends, before the traffic lights on the right side of the macOS titlebar — or left of the drag region end if there is no right-side space).
- Click: calls `togglePreferences()` directly. Identical to the `Cmd+,` path.
- No new Tauri event emitted. No new Rust code required.
- Keyboard shortcut `Cmd+,` is unchanged (D-17).

### Tab Rename (D-05)

- Double-click on File Tree tab label: no rename input. Title is fixed ("File Tree"). Double-click is a no-op.
- Double-click on GSD tab label: no rename input. Title is fixed ("GSD"). No-op.
- Double-click on Git Changes tab label: no rename input. Title is fixed ("Git Changes"). No-op.
- Double-click on Terminal / Agent / Editor tabs: existing rename-on-double-click behavior. Unchanged.
