**FileEntry struct:**

```rust
pub struct FileEntry {
    pub name: String,
    pub path: String,
    pub is_dir: bool,
}
```

**Commands to implement:**

1. `get_file_diff(path: String) -> Result&lt;String, String&gt;` (D-04)
   - Open git repo at path.parent() (the project dir)
   - Get DiffIndex between index and workdir
   - Filter to the specific file using DiffOptions
   - Use `diff.print(git2::DiffFormat::Patch, callback)` where callback builds unified diff string
   - In callback: `origin()` returns '+'/'-'/' ' char, `content()` returns bytes
   - Guard file size &gt; 1MB with error "File too large for diff viewing"
   - Use `spawn_blocking`

2. `list_directory(path: String) -> Result&lt;Vec&lt;FileEntry&gt;, String&gt;` (D-06)
   - Validate path: reject if contains ".." (path traversal attack mitigation)
   - `std::fs::read_dir` to get entries
   - For each entry: name (file_name().to_string_lossy()), path (entry.path().to_string_lossy()), is_dir (metadata.is_dir())
   - Sort entries: dirs first, then files, alphabetically
   - Use `spawn_blocking`

3. `read_file_content(path: String) -> Result&lt;String, String&gt;` (D-06)
   - Validate path: reject if contains ".." or not inside a project directory
   - `std::fs::read_to_string(&amp;path)`
   - Size guard: reject &gt; 1MB files
   - Use `spawn_blocking`

4. `write_checkbox(path: String, line: u32, checked: bool) -> Result&lt;(), String&gt;` (D-01)
   - Read all lines from path
   - Validate line index (0-based): line must exist
   - Parse the line: must match task list item regex `^\s*[-*]\s*\[[ x]\]`
   - Replace `[x]` with `[ ]` or `[x]` based on checked param
   - Write all lines back (atomic: tmp + rename)
   - Use `spawn_blocking`

   - Simply call read_file_content internally

**Path:** src-tauri/src/file_watcher.rs (new file)

**Imports:**
```rust
use notify::RecursiveMode;

use std::path::PathBuf;
```

**Note:** The watcher watches the project directory (not the file itself) because editors do atomic saves (delete+rename) which would remove the watch on the file directly. This is the same pattern as theme/watcher.rs watches the config dir, not theme.json directly.

**No new command needed** -- start_md_watcher is called directly from lib.rs setup when project changes. Frontend calls it via `invoke('set_project_path', { path })` which stores the path and (re)starts the watcher.
