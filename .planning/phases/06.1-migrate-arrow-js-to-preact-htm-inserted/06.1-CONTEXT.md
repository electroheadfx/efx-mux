**Gathered:** 2026-04-08

<domain>
## Phase Boundary

Replace Arrow.js with Preact + full modern frontend toolchain (Vite + TypeScript + Tailwind 4) across the entire frontend. Big-bang rewrite of all 10 components + main.js + state-manager.js. All existing UI behavior must be preserved — same layout, same reactivity, same look. Eliminates Arrow.js reactive template bugs (writeExpressions stringification, static-vs-reactive footguns).

Scope expanded beyond original roadmap: this is not just a framework swap but a full frontend toolchain modernization.

<decisions>
## Implementation Decisions

### Migration Strategy
- **D-01:** Big-bang rewrite — all Arrow.js usage replaced in one phase. No incremental migration, no dual reactive systems. Arrow.js and Preact cannot coexist in the same import map slot.
- **D-02:** All 10 component files (`sidebar.js`, `main-panel.js`, `right-panel.js`, `diff-viewer.js`, `file-tree.js`, `fuzzy-search.js`, `gsd-viewer.js`, `project-modal.js`, `tab-bar.js`), plus `main.js` and `state-manager.js`, are rewritten as `.tsx` files.

### State Management
- **D-03:** Use `@preact/signals` for all state management. Global signals for app-wide state (sidebar collapsed, active project, projects list), component-local signals for local state. Drop-in replacement for Arrow.js `reactive()` with fine-grained reactivity.
- **D-04:** Signal pattern: `signal()` for mutable state, `computed()` for derived values, `effect()` for side effects. Replaces Arrow.js `reactive({})` property access pattern.

### Component API Pattern
- **D-05:** All components written as Preact function components in `.tsx` files with JSX syntax. No htm tagged templates — full TypeScript type checking in templates with IDE autocomplete.
- **D-06:** Props typed with TypeScript interfaces. Event handlers use standard Preact patterns (`onClick`, `onInput`, etc.). Replaces Arrow.js `@click` directive syntax.
- **D-07:** Conditional rendering uses standard JSX patterns (`{condition && <Component />}`, ternaries). Eliminates the Arrow.js `() =>` reactive wrapper footgun that caused template leak text.

### Build Setup (Vite + Tauri)
- **D-08:** Vite as build tool and dev server. `npm run tauri dev` runs Vite dev server on localhost:1420 with Tauri webview pointing to it. HMR for instant component updates during development.
- **D-09:** Production: `npm run tauri build` runs Vite build to `dist/`, Tauri bundles `dist/` into `.app`. Remove `src/vendor/` directory entirely and inline importmap from `index.html`.
- **D-10:** npm dependencies via `package.json`: `preact`, `@preact/signals`, `tailwindcss@4`, `@tailwindcss/vite`, `typescript`, `vite`. No more vendored ESM files.

### CSS / Tailwind 4
- **D-11:** Convert existing `theme.css` and `layout.css` to Tailwind 4 utility classes in JSX. Define Solarized Dark palette as Tailwind theme tokens using CSS-first `@theme` syntax.
- **D-12:** Tailwind theme tokens: `--color-bg: #282d3a`, `--color-bg-raised: #363b3d`, `--color-border: #3e454a`, `--color-text: #8d999a`, `--color-text-bright: #92a0a0`, `--color-accent: #258ad1`.
- **D-13:** All component styling uses Tailwind utility classes. No separate CSS files for component styles. `xterm.css` remains as vendor CSS (xterm.js requirement).

### Claude's Discretion
- Vite config details (aliases, plugin order)
- TypeScript strictness level (strict vs relaxed)
- File/folder reorganization within `src/` (e.g., `src/components/` → `src/components/*.tsx`)
- Tailwind utility class choices for specific components
- Whether to keep `drag-manager.js` as-is or convert to TypeScript (utility, not a component)

</decisions>

### Established Patterns
- All components export a function that returns an Arrow.js `html` template. Migration: export Preact function component returning JSX.
- Global state via `reactive({})` in module scope. Migration: `signal()` exports from a shared store module.
- Event communication via `CustomEvent` dispatch on `document`. Migration: keep this pattern or replace with signals/context where appropriate.
- Tauri IPC via `window.__TAURI__.core.invoke()`. Migration: import from `@tauri-apps/api/core` via npm.

### Integration Points
- `src-tauri/tauri.conf.json` — Must update `devUrl` to Vite dev server URL and `frontendDist` to Vite output directory
- `src/index.html` — Complete rewrite: remove importmap, add Vite script entry point
- `package.json` — Currently minimal, needs full npm dependency setup
- Rust backend unchanged — all Tauri commands stay the same, only frontend calling conventions change
