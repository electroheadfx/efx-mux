<objective>
Close UAT Test 5: the user reports "Cmd+Backspace do not trigger delete modal". Plain Delete key works; only Cmd+Backspace fails.

Root cause (per debug session): WKWebView on macOS intercepts Command-modifier keystrokes via the AppKit NSResponder chain (doCommandBySelector:) BEFORE dispatching them as DOM keydown events. Cmd+Backspace maps to the NSResponder deleteBackward:/deleteToBeginningOfLine: selectors and is consumed before reaching JavaScript. This is a well-known WKWebView behavior (WebKit Bugzilla #191768 and multiple framework reports). Plain Delete is not bound to an NSResponder text-editing selector and reaches the handler normally — explaining the asymmetry the user sees.

The in-code keydown handlers at file-tree.tsx:1084-1094 (flat) and 1152-1158 (tree) are logically correct (a probe test confirmed they work when the event actually arrives). They just never receive the event.

**Compounding bug:** The existing test at file-tree.test.tsx:138-148 claims to verify Delete-key behavior but only passes by accident — the assertion `expect(document.body.textContent).toMatch(/Delete/)` is satisfied by the literal string "Delete" left in the DOM by earlier describe blocks that rendered context-menu items. Run in isolation, the test FAILS. So the plan's claim "Cmd+Backspace triggers the same delete flow on macOS" was never actually verified.

**Fix (per debug session fix_rationale option a):** Route Cmd+Backspace through Tauri's native menu. Register a hidden MenuItem with accelerator `CmdOrCtrl+Backspace`; `on_menu_event` emits a `delete-selected-tree-row` CustomEvent (via AppHandle emit); file-tree.tsx listens and calls `triggerDeleteConfirm(selectedEntry)`. Tauri intercepts the keystroke at the AppKit level BEFORE WebKit's doCommandBySelector: fires, bypassing the WKWebView suppression. Same pattern used for Cmd+W (closeUnifiedTab) and Cmd+, (togglePreferences).

Purpose: Restore user's muscle memory for the canonical macOS delete shortcut. Also hardens test coverage with a real assertion on the modal copy.
Output: One MenuItem + on_menu_event branch in Rust. One Tauri listen block in file-tree.tsx. Two test-file changes: replace the bogus Delete-key test, add a delete-selected-tree-row test.
</objective>
