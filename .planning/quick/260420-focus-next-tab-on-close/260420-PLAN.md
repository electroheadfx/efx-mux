# Quick Task: Focus Next Tab on Close

## Problem

When closing tabs (especially GSD, File Tree, or right-scope terminals), the code sets `activeTabId.value = ''` without switching to an adjacent tab first. This leaves the UI with no active tab even when other tabs exist in the same scope.

## Root Cause

In `closeUnifiedTab` (unified-tab-bar.tsx):
- Main-scope tabs correctly call `switchToAdjacentTab(tabId)` before closing
- GSD tabs (lines 1082-1083): set `activeTabId = ''` directly
- File Tree tabs (lines 1097-1098): set `activeTabId = ''` directly  
- Right-scope terminals: only call `switchToAdjacentTab` when it's the main-scope active (incorrect condition)

The existing `switchToAdjacentTab` only works with `allTabs` (main-scope view), not scope-local tabs.

## Solution

1. Add `switchToAdjacentTabInScope(scope, currentId)` helper that finds adjacent tab using scope-local tab order
2. Update GSD, File Tree, and right-scope terminal close paths to call it before setting `activeTabId = ''`
