**In src-tauri/src/git_ops.rs:**

1. Add a serializable struct for log entries:
```rust
#[derive(Debug, Clone, serde::Serialize)]
pub struct GitLogCommit {
    pub hash: String,        // full SHA
    pub short_hash: String,  // first 7 chars
    pub message: String,     // first line only (summary)
    pub author: String,
    pub timestamp: i64,      // Unix epoch seconds
    pub refs: Vec<String>,   // branch/tag names decorating this commit
}
```

2. Add `get_git_log_impl(repo_path: &str, limit: usize) -> Result<Vec<GitLogCommit>, GitError>`:
   - `Repository::open(repo_path)` (map NotARepo)
   - `repo.revwalk()` starting from HEAD (`revwalk.push_head()`)
   - Set sorting to `git2::Sort::TIME`
   - Iterate up to `limit` OIDs, for each:
     - `repo.find_commit(oid)` to get the commit object
     - Extract `commit.id()` (full + short hash via `&hash[..7]`)
     - Extract `commit.summary().unwrap_or("")` for message (first line)
     - Extract `commit.author().name().unwrap_or("Unknown")` for author
     - Extract `commit.time().seconds()` for timestamp
   - After collecting commits, resolve refs by iterating `repo.references()`:
     - For each reference, check if `reference.peel_to_commit()` matches any collected commit hash
     - Use `reference.shorthand()` for the display name (strips `refs/heads/`, `refs/remotes/`, etc.)
     - Append matching ref names to the corresponding commit's `refs` vec
   - Return the Vec<GitLogCommit>

3. Add the async Tauri command wrapper following existing pattern:
```rust
#[tauri::command]
pub async fn get_git_log(repo_path: String, limit: Option<usize>) -> Result<Vec<GitLogCommit>, String> {
    let max = limit.unwrap_or(50);
    spawn_blocking(move || get_git_log_impl(&repo_path, max))
        .await.map_err(|e| e.to_string())?.map_err(|e| e.to_string())
}
```
