  <name>Task 2: Implement tree mode with collapsible folders and indentation</name>
  <files>src/components/file-tree.tsx</files>
  <action>
Add tree mode rendering to file-tree.tsx. The tree mode uses the same `list_directory` backend but manages expanded state client-side.

1. **Tree state management:**
   ```typescript
   interface TreeNode {
     entry: FileEntry;
     children: TreeNode[] | null; // null = not loaded, [] = loaded but empty
     expanded: boolean;
     depth: number;
   }
   const treeNodes = signal<TreeNode[]>([]);
   ```
   - When tree mode activates, load the project root directory into `treeNodes` with depth=0
   - Each TreeNode wraps a FileEntry and tracks expanded state + loaded children

2. **Lazy loading of subdirectories:**
   - When a folder is clicked (or expanded via chevron), call `list_directory` for that folder's path
   - Store results as `children` on the TreeNode
   - Toggle `expanded` to show/hide children
   - If already loaded, just toggle `expanded` (no re-fetch)

3. **Chevron icons for folders:**
   - Create a small `ChevronIcon` component: inline SVG, 10x10, rotates 90deg when expanded
   - Collapsed: chevron points right (>). Expanded: chevron points down (v)
   - Color: `colors.textDim`, changes to `colors.textMuted` on hover
   - Only folders get chevrons. Files get no chevron (but maintain alignment with left padding)

4. **Indentation:**
   - Each depth level adds 16px left padding
   - So depth 0 = 12px padding (base), depth 1 = 28px, depth 2 = 44px, etc.
   - Formula: `paddingLeft: 12 + (node.depth * 16)`

5. **Flatten tree for rendering:**
   - Create a `flattenTree(nodes: TreeNode[]): TreeNode[]` function that recursively flattens expanded nodes into a render list
   - Only include children of expanded folders
   - This flat list is what gets rendered (map over it like the current entries list)

6. **Tree mode rendering:**
   - Replace the entries.value.map() with a conditional: if viewMode is 'flat', render current flat list; if 'tree', render flattened tree
   - Each tree row: chevron (folder only) + icon (FolderIcon/FileCodeIcon/FileTextIcon) + name + size
   - Clicking a folder toggles expand/collapse (does NOT drill down)
   - Clicking a file dispatches file-opened event (same as flat mode)
   - Selection highlight works the same way (selectedIndex over the flattened list)

7. **Keyboard navigation in tree mode:**
   - ArrowUp/ArrowDown: move through flattened list
   - Enter on folder: toggle expand/collapse
   - Enter on file: open file (dispatch event)
   - ArrowRight on collapsed folder: expand it
   - ArrowLeft on expanded folder: collapse it
   - ArrowLeft on file or collapsed folder: move to parent folder
   - Backspace: no-op in tree mode (tree doesn't drill down)

8. **Tree initialization:**
   - When viewMode switches to 'tree', load project root into treeNodes
   - When project changes (project-changed event), reload tree if in tree mode
   - The `currentPath` signal is still used in flat mode; tree mode ignores it

Style all tree elements consistently with the existing flat mode styling (same font sizes from fileTreeFontSize signal, same line heights from fileTreeLineHeight signal, same colors from tokens).
