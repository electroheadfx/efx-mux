In `createTerminal()`, inside the existing `attachCustomKeyEventHandler` callback (around line 44), add a new Cmd+K handler BEFORE the existing Cmd+Left handler:

```typescript
// Cmd+K -> clear terminal scrollback (standard macOS shortcut)
if (ev.metaKey && !ev.ctrlKey && !ev.altKey && (ev.key === 'k' || ev.key === 'K')) {
  ev.preventDefault();
  terminal.clear();
  return false;
}
```

This uses `terminal.clear()` which clears the scrollback buffer but preserves the current viewport line (same behavior as iTerm2 and VS Code integrated terminal). Both the main terminal and the right-panel bash terminal are created via `createTerminal()`, so this single change covers both.

Do NOT send `\x0b` (Ctrl+K) or shell `clear` command -- `terminal.clear()` is the correct xterm.js API for client-side scrollback clearing without shell interaction.
