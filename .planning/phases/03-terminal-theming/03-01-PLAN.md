1. **Create `src-tauri/src/theme/watcher.rs`** with:
   - Import `notify_debouncer_mini::{new_debouncer, DebounceEventResult}`, `std::time::Duration`, `tauri::Emitter`
   - Import `super::types::{theme_path, ThemeConfig}`
   - Function `pub fn start_theme_watcher(app_handle: tauri::AppHandle)`:
     - Get `theme_path()` and store as `target_path: PathBuf`
     - Get `config_dir()` as `watch_dir: PathBuf` (RESEARCH Pitfall 1: watch parent dir, NOT the file itself, because editors do atomic save via delete+rename)
     - Clone `app_handle` and `target_path` for the closure
     - Spawn a `std::thread::spawn` (NOT tokio -- matches Phase 2 PTY pattern: dedicated OS thread for blocking work)
     - Inside thread, create debouncer via `new_debouncer(Duration::from_millis(200), move |res: DebounceEventResult| { ... })` (per D-11: 200ms debounce)
     - In the debouncer callback:
       - `if let Ok(events) = res` -- ignore errors silently
       - Check if any event path matches `target_path` (filter for theme.json specifically)
       - If match: read file with `std::fs::read_to_string(&target_path)`
       - On read success: parse with `serde_json::from_str::<serde_json::Value>(&content)`
         - On parse success: emit `app_handle.emit("theme-changed", theme_value)` (per D-09). Use `serde_json::Value` as payload type (already Clone + Serialize per RESEARCH Pitfall 5)
         - On parse error (D-12): `eprintln!("[efxmux] Invalid theme.json: {}. Keeping current theme.", e)` -- do NOT emit event, keep current theme
       - On read error: `eprintln!("[efxmux] Failed to read theme.json: {}", e)`
     - After debouncer creation, call `debouncer.watcher().watch(&watch_dir, notify::RecursiveMode::NonRecursive)` (per RESEARCH: NonRecursive on parent dir)
     - Keep thread alive with `loop { std::thread::sleep(Duration::from_secs(3600)); }` (debouncer drops if scope exits)

2. **Update `src-tauri/src/lib.rs` setup() hook:**
