**unified-tab-bar.tsx -- Fix drag-and-drop (UAT Gap 1):**

The bug: `handleDrop` operates on `tabOrder` signal, but `tabOrder` only contains non-terminal tab IDs. Terminal tab IDs are never added to `tabOrder` (they're rendered first in `getOrderedTabs` by convention). So when dragging a terminal tab, `sourceIdx` or `targetIdx` is -1 and the function bails silently.

Fix `handleDrop` to operate on the full ordered list from `getOrderedTabs()` instead of just `tabOrder`:

1. In `handleDrop`, replace the current logic with:
   - Get the current full ordered list: `const ordered = getOrderedTabs()`
   - Get `allIds = ordered.map(t => t.id)`
   - Find `sourceIdx = allIds.indexOf(sourceId)` and `targetIdx = allIds.indexOf(targetId)`
   - If either is -1, bail
   - Splice source out, insert before target
   - Call `setProjectTabOrder(allIds)` with the new full order (including terminal IDs)

2. Update `getOrderedTabs()` to respect a tab order that now includes terminal IDs:
   - Remove the special-case "terminals always first" logic
   - Instead, sort ALL tabs by their position in `tabOrder`
   - Tabs not in `tabOrder` get appended at the end (preserves backward compat)

3. Remove the green plus icon: in `handleDragStart`, set `e.dataTransfer.effectAllowed = 'move'` (already done). Also set `e.dataTransfer.setDragImage` if needed, but the main fix is ensuring `dropEffect = 'move'` in `handleDragOver` (already done). The green plus icon appears because the OS default for 'copy' -- verify `e.dataTransfer.dropEffect = 'move'` is being set correctly. In `handleDragOver`, the code sets `e.dataTransfer.dropEffect = 'move'` -- this is correct. The green plus appears when `effectAllowed` and `dropEffect` mismatch. Ensure both are 'move'.

**dropdown-menu.tsx -- Fix wrong action on click (UAT Gap 3):**

The bug: When user clicks on a menu item, the `onClick` handler on the `div[role="menuitem"]` fires `item.action()`. But if the click target is the icon element inside the div (e.g., `<item.icon size={14} />`), the event might not propagate correctly to the div's onClick in some cases, or the icon could intercept pointer events.

Fix:
1. Add `pointerEvents: 'none'` to the icon element so clicks pass through to the parent div:
   ```tsx
   {item.icon && <item.icon size={14} style={{ pointerEvents: 'none' }} />}
   ```
2. Add `pointerEvents: 'none'` to the label text span as well, ensuring the div is always the click target.
3. Alternatively, wrap the entire menuitem content in a single clickable container and ensure `onClick` is on the outermost element that has the action reference.
4. Also ensure the `selectedIndex` state is synchronized on click. Before calling `item.action()`, verify we're calling the correct item by using the `item` from the map closure directly (which it already does), not `selectableItems[selectedIndex]`.

The current code at line 207 already does `item.action()` directly from the map closure, which is correct. The icon `pointerEvents: 'none'` fix prevents the icon from stealing the click event.
