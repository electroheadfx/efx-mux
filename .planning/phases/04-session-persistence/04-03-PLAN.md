Add a Rust-side window close handler that saves state synchronously when the user closes the app window. This guarantees state.json is written before the process exits, regardless of whether the JS beforeunload async invoke completes.

**Approach:** Use Tauri 2's `Builder::on_window_event` to intercept `WindowEvent::CloseRequested`. On close, read the current state.json from disk (which the JS side has been updating throughout the session via invoke('save_state')), and if it exists, do nothing extra (the latest state is already on disk from the most recent drag/theme/session update). But to handle the edge case where the JS beforeunload invoke did not complete, we add a final synchronous save.

The problem is that the Rust side does not hold in-memory app state -- state lives in the JS frontend and is passed to Rust only via invoke('save_state'). On close, JS calls invoke('save_state') but that async call may not complete before teardown.

**Solution:** Use Tauri's managed state (`app.manage()`) to keep a copy of the latest AppState in Rust memory. Update it every time `save_state` is called. On window close, write the managed copy to disk synchronously.

**Step 1: Add managed state wrapper to state.rs.**

Add this import and type at the top of state.rs:
```rust
use std::sync::Mutex;

/// Tauri-managed wrapper for in-memory AppState.
/// Updated on every save_state call; written to disk on window close.
pub struct ManagedAppState(pub Mutex<AppState>);
```

**Step 2: Update the `save_state` Tauri command** to also update the managed state:

Change the save_state command signature to accept the Tauri managed state:
```rust
#[tauri::command]
pub async fn save_state(
    state_json: String,
    managed: tauri::State<'_, ManagedAppState>,
) -> Result<(), String> {
    let state: AppState = serde_json::from_str(&state_json).map_err(|e| e.to_string())?;
    // Update in-memory copy for the close handler
    if let Ok(mut guard) = managed.0.lock() {
        *guard = state.clone();
    }
    tauri::async_runtime::spawn_blocking(move || save_state_sync(&state))
        .await
        .map_err(|e| e.to_string())?
}
```

**Step 3: Update lib.rs** to manage state and add the close handler.

Add these changes to `src-tauri/src/lib.rs`:

1. Add import for the managed state type:
```rust
use state::{get_config_dir, load_state, save_state, ManagedAppState};
```

2. In `setup()`, after `state::ensure_config_dir()`, load initial state and manage it:
```rust
// Load initial state into Tauri managed state (for close handler)
let initial_state = state::load_state_sync();
app.manage(ManagedAppState(std::sync::Mutex::new(initial_state)));
```

3. Add `.on_window_event` BEFORE `.run()` on the Builder chain (after `.invoke_handler()`):
```rust
.on_window_event(|window, event| {
    if let tauri::WindowEvent::CloseRequested { .. } = event {
        // Synchronously save the latest in-memory state to disk.
        // This guarantees state.json is written even if the JS
        // beforeunload async invoke did not complete (WR-03 fix).
        if let Some(managed) = window.try_state::<ManagedAppState>() {
            if let Ok(guard) = managed.0.lock() {
                if let Err(e) = state::save_state_sync(&guard) {
                    eprintln!("[efxmux] WARNING: Failed to save state on close: {}", e);
                }
            }
        }
    }
})
```

The full Builder chain in lib.rs should read:
```rust
tauri::Builder::default()
    .setup(|app| {
        // ... existing setup code ...
        // After state::ensure_config_dir():
        let initial_state = state::load_state_sync();
        app.manage(state::ManagedAppState(std::sync::Mutex::new(initial_state)));
        // ... rest of setup ...
        Ok(())
    })
    .invoke_handler(tauri::generate_handler![
        spawn_terminal,
        write_pty,
        resize_pty,
        ack_bytes,
        load_theme,
        import_iterm2_theme,
        load_state,
        save_state,
        get_config_dir,
    ])
    .on_window_event(|window, event| {
        if let tauri::WindowEvent::CloseRequested { .. } = event {
            if let Some(managed) = window.try_state::<ManagedAppState>() {
                if let Ok(guard) = managed.0.lock() {
                    if let Err(e) = state::save_state_sync(&guard) {
                        eprintln!("[efxmux] WARNING: Failed to save state on close: {}", e);
                    }
                }
            }
        }
    })
    .run(tauri::generate_context!())
    .expect("error while running tauri application")
```

**Key details:**
- `ManagedAppState` uses `Mutex<AppState>` (not async -- the close handler runs synchronously)
- `save_state` command now takes `tauri::State<'_, ManagedAppState>` -- Tauri auto-injects managed state into commands
- The `on_window_event` closure receives `(&Window, &WindowEvent)` in Tauri 2
- `try_state::<ManagedAppState>()` is used instead of `state::<>()` to avoid panicking if state is not managed
- `save_state_sync` is called directly (not via spawn_blocking) since the close handler is synchronous and we need to block until the write completes
- The JS beforeunload handler remains as a belt-and-suspenders approach -- it may succeed, but the Rust handler guarantees the save
