# Reference Sidebar visual patterns
- Header: flex items-center justify-between, px-4 pt-4 pb-3, EFXMUX text (13px, semibold, tracking-3px, textPrimary), + button (w-6 h-6 rounded-6 accent bg)
- Project items: gap-10, px-16 py-[10px], mx-2, rounded-6, active has bgElevated + borderLeft 3px accent
- Status dot: 8x8 circle, statusGreen if active, textDim if not
- Project name: 13px, font-medium if active, textPrimary/textMuted, truncated
- Branch: text-[10px] mono, accent if active, textDim if not
- Section labels: GeistMono 10px uppercase letterSpacing-1.5px, textDim
- Git badges: 18x18 rounded-3, statusYellowBg for M, statusMutedBg for U
- Collapsed: 40px icon strip, each icon has aria-label

# Key decision points for this rewrite
1. The reference Sidebar uses hardcoded project data. Production sidebar uses signals (projects, activeProjectName, gitData, gitFiles).
2. The reference has ProjectItem sub-component. Production has ProjectRow and CollapsedIcon sub-components.
3. Production has RemoveDialog as a modal overlay. Reference has no remove dialog.
4. Production has git change badges (M/S/U) and branch display. Reference has GitFile sub-component.

Strategy: Keep all Production sub-components and logic, only update visual styles. Merge reference visual patterns (color values, padding, radius) into existing component structure.
