  <name>Task 1: Full Phase 22 UAT — dynamic tabs, vertical split, cross-scope drag, preferences button, persistence</name>
  <what-built>
Complete Phase 22 feature set:
- Sticky tab kind removed from UnifiedTabBar; File Tree, GSD, Git Changes now render as uniform dynamic tabs with fixed titles and × close buttons (D-01, D-05).
- TerminalScope expanded from 'main' | 'right' to 6 hierarchical ids ('main-0'..'main-2', 'right-0'..'right-2') (D-10).
- Shared per-project PTY session-name counter — names stay stable on drag, collisions prevented by construction (D-12).
- Main + Right zones support 1..3 stacked sub-panes via split icon in every UnifiedTabBar; 3-cap enforced with disabled state and 'maximum 3 panes reached' aria-label (D-07, D-08).
- Intra-zone `.split-handle-h-intra` resize handle (`ns-resize` cursor); ratios persist per zone per handle in state.layout (D-09).
- All tab kinds cross-scope draggable (Terminal, Agent, Editor, GSD, Git Changes, File Tree) between any two scopes; drop affordance `.drop-target` outline on target tab bar during drag, no pane body tint (D-11, D-13, D-14).
- GSD + Git Changes enforced as singletons — `+` menu item dimmed with 'Already open in another pane' hint when open elsewhere (D-04).
- File Tree is non-singleton — every scope can hold its own; each body renders the same project tree with independent UI state.
- Preferences button in titlebar right side with Lucide `Settings` icon 14px; click calls `togglePreferences()`; Cmd+, path unchanged (D-15, D-16, D-17, D-18).
- Empty scope shows 'No tabs open' / 'Press + to add a tab.' placeholder (D-03).
- Legacy state.json migrates silently: `terminal-tabs:<project>` → `:main-0`, `right-terminal-tabs:<project>` → `:right-0`, sticky ids 'file-tree' / 'gsd' in persisted activeTabId dropped (D-06, D-10).
- First-launch defaults (D-02): main-0 auto-spawns Terminal-1; right-0 auto-spawns GSD + File Tree as dynamic tabs.
  </what-built>
  <how-to-verify>
Open the app (user runs `pnpm tauri dev` on their side — DO NOT invoke from the executor). Walk through each group below and tick boxes.

**Group 1 — Uniform dynamic tabs (TABS-01)**
1. [ ] On first launch with a fresh project (or freshly deleted `state.json`), the main zone shows one terminal tab (Terminal-1) and the right zone shows both GSD and File Tree as tabs (not as sticky segments — confirm by presence of × close buttons on hover over each).
2. [ ] Hover over File Tree tab → × close button visible. Click × → tab closes; right zone now shows only GSD tab.
3. [ ] Open `+` menu in right-0 tab bar → File Tree entry is enabled. Click it → File Tree tab reappears in right-0.
4. [ ] Double-click File Tree tab label → NO rename input appears. Same test for GSD tab. Same for Git Changes tab (open it via `+`).
5. [ ] Double-click a terminal tab label → rename input DOES appear (regression check).
6. [ ] Drag File Tree tab to reorder inside right-0 (mouse-based drag — hold + move sideways) → insertion indicator appears, tab reorders on drop.

**Group 2 — Singleton enforcement (TABS-01 subset)**
1. [ ] Open `+` menu in main-0 tab bar (initial state). "GSD" entry is DIMMED (aria-disabled) because GSD is open in right-0.
2. [ ] Close the GSD tab in right-0 via × → main-0's `+` menu now shows "GSD" as ENABLED.
3. [ ] Click "GSD" in main-0's `+` menu → GSD tab appears in main-0; right-0's `+` menu now dims "GSD".
4. [ ] Open `+` menu → hover the dimmed "GSD" entry → optional tooltip "Already open in another pane" (if implemented per UI-SPEC copywriting).
5. [ ] Same rules for Git Changes.
6. [ ] File Tree entry in `+` menu is NEVER dimmed (non-singleton) — open File Tree in both main-0 and right-0 simultaneously. Both bodies render (same tree, independent scroll/expand).

**Group 3 — Vertical split (SPLIT-01, SPLIT-02)**
1. [ ] Right of the `+` menu trigger in every tab bar: a split-icon button (Lucide `Rows2`). Hover → accent color.
2. [ ] Click split icon in main-0 → new sub-scope `main-1` appears below main-0 with its own empty tab bar and `.scope-empty-placeholder` showing "No tabs open" / "Press + to add a tab."
3. [ ] Click split icon in main-1 → third sub-scope `main-2` appears. All three stacked vertically.
4. [ ] Click split icon a fourth time in any main sub-scope → NO new scope appears; icon shows disabled state (opacity 0.4, cursor `not-allowed`, aria-label "Split pane (maximum 3 panes reached)").
5. [ ] Hover between main-0 and main-1 → cursor becomes `ns-resize`. Drag down → main-0 grows, main-1 shrinks. Minimum height respected (scope does not collapse below ~48px).
6. [ ] Restart app (close + reopen). Sub-scope count and resize ratios persist — main zone still shows 3 sub-scopes at the dragged ratios.
7. [ ] Repeat for right zone (right-0 → right-1 → right-2 → 3-cap).

**Group 4 — Cross-scope drag (SPLIT-03, SPLIT-04)**
1. [ ] Start with a terminal tab in main-0. Drag it onto right-0's tab bar. During drag: `.drop-target` outline (accent border) appears around right-0's tab bar. Insertion-slot line appears at drop position. Pane body does NOT tint.
2. [ ] Drop → terminal tab moves to right-0. PTY session keeps running (check: terminal contents + scrollback intact). No `[exited]` state.
3. [ ] Check `tmux ls` (from another terminal outside Efxmux): the session name for that tab is UNCHANGED (e.g. still `<project>-2`).
4. [ ] Drag the same terminal from right-0 back to main-1. Session name still unchanged.
5. [ ] Repeat drag test for: Agent tab (no agent-quit confirm should fire on drag — only on close), Editor tab (ownerScope flips), GSD tab (singleton — moves entirely, source + target `+` menus re-dim correctly), Git Changes tab (same), File Tree tab (ownerScope flips, scope can still have its own File Tree via `+` menu — verify multi-File-Tree state).
6. [ ] Drag a tab across three scopes in quick succession (main-0 → right-1 → main-2) — session name remains stable at each hop.
7. [ ] Create 5 terminal tabs across different scopes — session names are `<project>`, `<project>-2`, `<project>-3`, `<project>-4`, `<project>-5` (shared counter, no duplicates).

**Group 5 — Preferences button (PREF-01)**
1. [ ] Titlebar right side (overlay zone): a `Settings` gear icon button. Mouse-hover → accent color border.
2. [ ] Click button → Preferences panel slides in.
3. [ ] Press `Cmd+,` keyboard shortcut → Preferences panel toggles (same behavior as button).
4. [ ] Use "Efxmux → Preferences…" in the macOS menu → same behavior.
5. [ ] Try to drag the window by clicking + dragging AT the Preferences button → window should NOT drag (button has no-drag). Click elsewhere in titlebar → window drags normally.

**Group 6 — Empty scope behavior (D-03)**
1. [ ] Close all tabs in main-1 → `.scope-empty-placeholder` renders with heading "No tabs open" and body "Press + to add a tab.".
2. [ ] Click + in the empty scope's tab bar → + menu opens; click a terminal entry → new terminal tab spawns in that scope.

**Group 7 — Legacy state migration (D-06, D-10)**
Only run if you have a saved Phase 20 state.json:
1. [ ] Pre-migrate, copy your current `~/.config/efxmux/state.json` as backup.
2. [ ] Start app with the legacy state → no error; tabs appear in main-0 and right-0 matching pre-migration layout.
3. [ ] Inspect state.json after first save → legacy keys (`terminal-tabs:<project>`, `right-terminal-tabs:<project>`) are gone; new keys (`terminal-tabs:<project>:main-0`, `terminal-tabs:<project>:right-0`) present.
4. [ ] Persisted activeTabId `'file-tree'` / `'gsd'` is reset to `''`; post-first-render, D-02 defaults reapply so the same visual result is preserved.

**Group 8 — WebGL stress test (Pitfall 1)**
1. [ ] Spawn 6 scopes (3 main + 3 right) with a terminal tab in each + one extra tab per scope. Use all of them briefly (type in each).
2. [ ] Check browser devtools console (right-click → Inspect Element → Console) for WebGL context-loss warnings.
3. [ ] If context-loss fires, terminal(s) fall back to DOM renderer — visually slightly different but still functional. Log any observed behavior in the summary.
