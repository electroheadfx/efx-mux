# Requirements: Efxmux

**Defined:** 2026-04-12
**Core Value:** Developers using Claude Code or OpenCode lose context switching between the terminal, their editor, and their planning docs. Efxmux collapses all of that into one native window with a terminal-first aesthetic -- dark, fast, keyboard-driven.

## Out of Scope

| Feature | Reason |
|---------|--------|
| E2E browser tests | High complexity, Tauri webview automation fragile |
| PTY integration tests | Requires real PTY + tmux, slow, already battle-tested from daily use |
| Snapshot testing | Brittle, low signal for this codebase |
| Visual regression testing | No visual regression tool warranted at this scale |
| 100% coverage target | Diminishing returns; sensible coverage for critical paths only |
| xterm.js render testing | Terminal canvas/WebGL cannot instantiate in jsdom |
