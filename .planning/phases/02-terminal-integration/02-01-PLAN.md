**PtyState struct** (managed Tauri state):
```rust
pub struct PtyState {
    pub writer: Arc<Mutex<Box<dyn std::io::Write + Send>>>,
    pub master: Arc<Mutex<Box<dyn portable_pty::MasterPty + Send>>>,
    pub sent_bytes: Arc<AtomicU64>,
    pub acked_bytes: Arc<AtomicU64>,
    pub slave: Arc<Mutex<Option<Box<dyn portable_pty::SlavePty + Send>>>>,
}
```
- `writer`: Arc<Mutex<>> because take_writer() is one-shot (per CLAUDE.md gotcha)
- `slave`: kept alive in Option until child exits (per CLAUDE.md gotcha)
- `master`: wrapped in Arc<Mutex<>> for resize access from command thread

**check_tmux() function** (per D-01):
```rust
pub fn check_tmux() -> Result<String, String>
```
- Runs `std::process::Command::new("tmux").arg("-V").output()`
- Returns Ok(version_string) on success
- Returns Err("tmux not found. Install with: brew install tmux") on failure

**spawn_terminal command** (per D-03, D-04, D-05, D-09, D-10):
```rust
#[tauri::command]
pub async fn spawn_terminal(
    app: tauri::AppHandle,
    on_output: tauri::ipc::Channel<Vec<u8>>,
    session_name: String,
) -> Result<(), String>
```
- Sanitize session_name: strip non-alphanumeric/hyphen/underscore chars (security: prevent shell injection)
- Call `portable_pty::native_pty_system()` to get PtySystem
- Open PTY with `PtySize { rows: 24, cols: 80, pixel_width: 0, pixel_height: 0 }`
- Build command: `CommandBuilder::new("tmux")` with args `["new-session", "-A", "-s", &sanitized_name]`
- Spawn child via `pair.slave.spawn_command(cmd)`
- Take writer via `pair.master.take_writer()` -- store in Arc<Mutex<>>
- Clone reader via `pair.master.try_clone_reader()`
- Create AtomicU64 counters for sent_bytes and acked_bytes
- Store PtyState via `app.manage(PtyState { ... })`
- Spawn dedicated OS thread (`std::thread::spawn`, NOT tokio::spawn -- per Research Pitfall 5) for read loop:
  - Buffer: `vec![0u8; 4096]` (Claude's discretion: 4KB read buffer)
  - Loop: check unacknowledged bytes (`sent - acked`). If > 400_000 (HIGH watermark per D-09), sleep 10ms and continue (backpressure pause)
  - Read from PTY master reader into buffer
  - On Ok(0): break (EOF)
  - On Ok(n): clone chunk `buf[..n].to_vec()`, increment sent_bytes by n, send via `on_output.send(chunk)`
  - On Err: break
- Return Ok(())

**write_pty command:**

**resize_pty command** (per D-13, D-14):

#[tauri::command]
pub fn resize_pty(cols: u16, rows: u16, state: tauri::State<'_, PtyState>) -> Result<(), String>

- Lock master mutex, call `master.resize(PtySize { rows, cols, pixel_width: 0, pixel_height: 0 })`

**ack_bytes command** (per D-11):

- Return Ok(())

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JS -> Rust IPC | Frontend invokes Tauri commands with user-controlled parameters (session_name, data, cols, rows, count) |
| Rust -> tmux subprocess | Session name passed to `std::process::Command` args |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | spawn_terminal session_name | mitigate | Sanitize session_name: strip all chars except alphanumeric, hyphen, underscore before passing to tmux Command args. Prevents shell injection via crafted session names. |
| T-02-02 | Denial of Service | PTY read loop unbounded memory | mitigate | Flow control watermarks: pause PTY reads at 400KB unacknowledged bytes (D-09), resume at 100KB (D-11). Caps memory usage. |
| T-02-03 | Denial of Service | resize_pty with extreme values | accept | cols/rows are u16 (max 65535). Pathological values cause PTY issues but no security impact. Local desktop app, single user. |
| T-02-04 | Information Disclosure | PTY output via Channel | accept | All output goes to same-origin webview. No cross-origin risk. Local desktop app. |
</threat_model>
