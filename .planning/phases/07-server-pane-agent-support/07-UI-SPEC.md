## Component Inventory

### Server Pane (3-state collapsible)

**States:**
| State | Height | Content Visible | Toolbar Visible |
|-------|--------|----------------|-----------------|
| Strip (default) | 28px | No | Yes (compact: title + buttons inline) |
| Expanded | Persisted (default 200px, min 100px, max 50% of main panel) | Yes (scrollable log area) | Yes |
| Collapsed | 0px | No | No |

**Cycle:** Ctrl+\` -> strip -> expanded -> collapsed -> strip (D-01)

**Split handle:** Visible only when expanded. Reuses `.split-handle-h` class (already exists in app.css). Drag-resize updates persisted height in state.json (D-02, D-03).

### Server Toolbar

**Layout:** Single horizontal band, `display: flex`, `align-items: center`, `justify-content: space-between`. Left side: status dot + "Server" label. Right side: button group.

**Status indicator:** 8px circle (border-radius: 50%) to the left of "Server" label.
| Server State | Dot Color | Dot CSS |
|-------------|-----------|---------|
| Stopped | `--color-text` at 40% opacity | `bg-text/40` |
| Running | `#859900` | `bg-[#859900]` |
| Crashed | `#dc322f` | `bg-[#dc322f]` |

### Server Buttons

**Shared button style:**
```
background: var(--color-bg-raised)
border: 1px solid var(--color-border)
color: var(--color-text)
padding: 2px 8px
border-radius: 2px
font-size: 11px
font-weight: 600
font-family: system-ui, sans-serif
cursor: pointer
transition: all 0.15s ease
```

**Hover (enabled):** `border-color: var(--color-accent)`, `color: var(--color-text-bright)`

**Disabled:** `opacity: 0.4`, `cursor: not-allowed`, no hover effect

**Button states by server status:**

| Server Status | Start | Stop | Restart | Open |
|--------------|-------|------|---------|------|
| Stopped | enabled | disabled | disabled | disabled |
| Running | disabled | enabled | enabled | enabled (if URL detected) |
| Crashed | enabled | disabled | disabled | disabled |
| Unconfigured | disabled | disabled | disabled | disabled |

### Server Log Area

**Container:** `overflow-y: auto`, `padding: 16px`, `font-family: FiraCode`, `font-size: 12px`, `line-height: 1.5`, `white-space: pre-wrap`, `word-break: break-all`.

**Scrollback limit:** 5000 lines. When exceeded, remove oldest lines from the DOM.

**ANSI color mapping:** Use project Solarized Dark palette (from RESEARCH.md Pattern 2):
| ANSI Code | Color |
|-----------|-------|
| Red (31) | `#dc322f` |
| Green (32) | `#859900` |
| Yellow (33) | `#b58900` |
| Blue (34) | `#268bd2` |
| Magenta (35) | `#d33682` |
| Cyan (36) | `#2aa198` |
| White (37) | `#eee8d5` |
| Bold (1) | `font-weight: bold` |
| Reset (0) | Close all open spans |

**Auto-scroll:** Log area auto-scrolls to bottom on new output UNLESS user has manually scrolled up. Detect via `scrollTop + clientHeight < scrollHeight - 4` (4px threshold).

### Agent Fallback Banner

**When:** Neither `claude` nor `opencode` binary found (AGENT-05).

**Rendering:** Written to xterm.js terminal via `term.writeln()` before bash session starts. Not an HTML overlay.

**Content:** 3 blank lines above, then the banner text, then 2 blank lines below, then bash prompt starts.

## Interaction Contract

### Ctrl+\` Toggle

| Trigger | Current State | Next State | Animation |
|---------|--------------|------------|-----------|
| Ctrl+\` | strip (28px) | expanded (persisted height) | Height transition 150ms ease |
| Ctrl+\` | expanded | collapsed (0px) | Height transition 150ms ease |
| Ctrl+\` | collapsed | strip (28px) | Height transition 150ms ease |

**Keyboard capture:** `document.addEventListener('keydown', handler, { capture: true })` to fire before xterm.js. Check `e.ctrlKey && e.key === '\`'`. Call `e.preventDefault()` and `e.stopPropagation()`.

### Split Handle Drag (expanded state only)

- Minimum server pane height: 100px
- Maximum server pane height: 50% of main panel container height
- On drag end: persist height to `state.json` via existing `updateLayout` pattern
- Split handle visibility: hidden when state is "strip" or "collapsed"

### Open in Browser

- Click "Open" button -> invoke `openUrl(server_url)` via `@tauri-apps/plugin-opener`
- If `server_url` not configured in project: use auto-detected URL from stdout (D-09)
- If no URL available: button stays disabled

### Stop Server (destructive)

- No confirmation dialog. Stop is a low-risk action (dev servers restart trivially).
- Stop sends SIGTERM to the process group. If process does not exit within 3 seconds, send SIGKILL.
- Log line appended: `[server] Stopped`

### Restart Server

- Equivalent to Stop + Start with no user confirmation.
- Log area is NOT cleared on restart. New session appended below with a visual separator: `--- Restarting ---`

## Layout Integration

The server pane lives inside `.main-panel` (already scaffolded in main-panel.tsx).

```
.main-panel (flex: column)
  |-- .terminal-area (flex: 1, min-height: 100px)
  |-- .split-handle-h (4px, visible only when expanded)
  |-- .server-pane (flex-shrink: 0, height: 28px | persisted | 0px)
       |-- .server-pane-toolbar (28px, flex row)
       |-- .server-pane-logs (flex: 1, overflow-y: auto, visible only when expanded)
```

**CSS additions needed in app.css:**

```css
.server-pane-toolbar {
  height: 28px;
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 0 8px;
  background: var(--color-bg-raised);
  border-bottom: 1px solid var(--color-border);
  flex-shrink: 0;
}

.server-pane-logs {
  flex: 1;
  min-height: 0;
  overflow-y: auto;
  padding: 4px 16px;
  font-family: var(--font-family-mono);
  font-size: 12px;
  line-height: 1.5;
  color: var(--color-text);
  white-space: pre-wrap;
  word-break: break-all;
}

.server-pane.state-strip {
  height: 28px;
  overflow: hidden;

.server-pane.state-expanded {
  height: var(--server-pane-h, 200px);

  height: 0;
  overflow: hidden;
  border-top: none;
}

.server-pane {
  transition: height 0.15s ease;
}
```
