In app.css, after the existing xterm scrollbar rules (line 146), add a global custom scrollbar block that applies to ALL scrollable containers in the app. This ensures sidebar, right panel viewers (GSD, Diff, File Tree), and file viewer all get themed scrollbars matching the xterm style:

```css
/* --- Global custom scrollbars (match xterm style) --- */
*::-webkit-scrollbar { width: 8px; height: 8px; }
*::-webkit-scrollbar-track { background: transparent; }
*::-webkit-scrollbar-thumb { background: var(--color-border); border-radius: 4px; }
*::-webkit-scrollbar-thumb:hover { background: var(--color-text); }
```

The track is transparent (instead of bg color) so it blends into whatever panel background it sits over. The thumb matches the existing xterm styling (border color, 4px radius, text color on hover).

Also add padding to the terminal area. In the existing `.terminal-area` rule (line 118-122), the main-panel.tsx already has `p-1` on the terminal-area div (line 62). That is fine for xterm. No CSS change needed for terminal padding -- it is already handled inline.

For the sidebar scrollbar, the `.sidebar-content` class already has `overflow-y: auto` (line 70) so the global scrollbar rules will apply automatically.

For right panel content areas, they use `overflow-hidden` on the wrapper and individual viewers handle their own scroll. The global rule covers all of them.

Remove the xterm-specific scrollbar rules (lines 142-146) since the global rules now cover them. The xterm viewport will inherit the global scrollbar styling. Actually -- KEEP the xterm-specific rules as they are more specific and ensure xterm's viewport (which uses a non-transparent track) stays correct. The global rules use transparent track which is better for app panels but xterm needs the bg-colored track to avoid visual artifacts. So keep both: global with transparent track, xterm-specific with bg track.
