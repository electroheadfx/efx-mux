# Quick 260415-h39: Add Git History Accordion Section Summary

Git log history with ref-decorated commit entries via git2 revwalk, displayed in a collapsible History section below Staged/Changes in the git control tab.

| 2 | Add History collapsible section to git-control-tab | 94ac94a | historySectionOpen/gitHistory signals, relativeTime helper, HistoryEntry component, scrollable area restructure |

## Implementation Details

### Rust Backend (git_ops.rs)
- `GitLogCommit` struct: hash, short_hash, message, author, timestamp (i64 unix), refs (Vec<String>)
- `get_git_log_impl`: opens repo, starts revwalk from HEAD sorted by TIME, collects up to `limit` commits, then iterates all references to decorate commits with matching shorthand ref names
- Async Tauri wrapper with spawn_blocking (consistent with existing pattern)

### Frontend Service (git-service.ts)
- `GitCommitEntry` interface mirroring Rust struct (distinct from existing GitLogEntry error log type)
- `getGitLog` wrapper returning empty array on failure (graceful degradation)

### UI Component (git-control-tab.tsx)
- `historySectionOpen` signal defaulting to `false` (collapsed)
- `gitHistory` signal populated during `refreshGitFiles` (mount, project-change, git-status-changed)
- `relativeTime` helper: <1m, Nm, Nh, Nd, or "MMM D" for older commits
- `HistoryEntry` component: mono commit message (truncated), accent-colored ref badge pills, dim author + relative time
- Scrollable area restructured from ternary to flat sections: empty state only when no files AND no history; Staged, Changes, History render independently
