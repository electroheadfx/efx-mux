**Gap 3 — Toolbar shows Start after restart (UAT test 6, MAJOR):**

Root cause: When `restartServer` is called, the Rust side calls `stop_server_inner` (kills old process) then `start_server` (spawns new process). The old process's waiter thread emits `server-stopped` with exit code 143 AFTER the frontend has already set `serverStatus = 'running'` for the new process. The `listenServerStopped` callback sees `exitCode >= 0` while status is 'running' and flips to 'crashed', which makes the toolbar show the Start button.

The exit-143-is-clean fix from Task 1 partially addresses this (since the old process exits with 143 from SIGTERM). However, there's still a race if the old process exits with a different code during restart (e.g., if it was already failing). Add a belt-and-suspenders `isRestarting` guard.

Fix in `src/components/server-pane.tsx`:

1. Add a module-level mutable flag (NOT a signal, no re-render needed):
```typescript
let isRestarting = false;
```

2. In `handleRestart`, set the flag BEFORE calling `restartServer`:
```typescript
const handleRestart = async () => {
  const proj = getActiveProjectEntry();
  if (!proj?.server_cmd) return;
  isRestarting = true;
  serverLogs.value = [...serverLogs.value, ansiToHtml('[server] --- Restarting ---\n')];
  serverStatus.value = 'running';
  try {
    await restartServer(proj.server_cmd, proj.path);
  } catch (err) {
    serverLogs.value = [...serverLogs.value, ansiToHtml(`[server] Restart failed: ${err}\n`)];
    serverStatus.value = 'crashed';
  } finally {
    // Clear flag after a delay to allow stale events to pass
    setTimeout(() => { isRestarting = false; }, 2000);
  }
};
```

3. In the `listenServerStopped` callback, skip state changes while restarting:
```typescript
listenServerStopped((exitCode: number) => {
  // Skip stale stopped events during restart window
  if (isRestarting) return;
  
  if (exitCode === 143 || exitCode === 137) {
    if (serverStatus.value === 'running') {
      serverStatus.value = 'stopped';
    }
    return;
  }
  if (exitCode >= 0 && serverStatus.value === 'running') {
    serverStatus.value = 'crashed';
    serverLogs.value = [...serverLogs.value, ansiToHtml(`[server] Process exited (code ${exitCode})\n`)];
  }
});
```

4. Also update `handleStop` to set `isRestarting = false` (in case user stops during a restart window):
```typescript
const handleStop = async () => {
  isRestarting = false;  // Cancel any restart guard
  serverLogs.value = [...serverLogs.value, ansiToHtml('[server] Stopped\n')];
  serverStatus.value = 'stopped';
  try { await stopServer(); } catch (err) { console.warn('[efxmux] Stop failed:', err); }
};
```

This double-guard approach (exit 143 = clean + isRestarting flag) ensures the toolbar always reflects the correct server state.
