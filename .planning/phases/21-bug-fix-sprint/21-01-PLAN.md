```typescript
useEffect(() => {
  let cancelled = false;
  const unlistenPromise = listen<string>('file-tree-changed', async (event) => {
    if (cancelled || !viewRef.current || !setupRef.current) return;
    // Path-filter: only react if the changed path matches THIS tab's file
    const changedPath = event.payload;
    if (changedPath !== filePath) return;

    try {
      const diskContent = await readFile(filePath);
      const savedContent = setupRef.current.getSavedContent();
      if (diskContent === savedContent) return; // No-op: file unchanged

      // Determine dirty state by comparing CURRENT editor doc to savedContent
      const currentDoc = viewRef.current.state.doc.toString();
      const isDirty = currentDoc !== savedContent;

      if (isDirty) {
        // D-07: do NOT clobber unsaved edits — flag the indicator instead.
        setEditorChangedOnDisk(tabId, true);
        return;
      }

      // Clean tab — reload in place. Preserve scroll best-effort.
      const view = viewRef.current;
      const scrollTop = view.scrollDOM.scrollTop;
      view.dispatch({
        changes: { from: 0, to: view.state.doc.length, insert: diskContent },
      });
      setupRef.current.setSavedContent(diskContent);
      setEditorDirty(tabId, false);
      setEditorChangedOnDisk(tabId, false);
      // Restore scroll after dispatch (next frame so layout settles)
      requestAnimationFrame(() => { view.scrollDOM.scrollTop = scrollTop; });
    } catch (err) {
      const msg = err instanceof Error ? err.message : String(err);
      if (/no such file|not found|notfound|os error 2/i.test(msg)) {
        closeEditorTabForDeletedFile(filePath);
      }
      // else: transient — ignore.
    }
  });
  return () => {
    cancelled = true;
    unlistenPromise.then(fn => fn());
  };
}, [filePath, tabId]);
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FS → Rust watcher → Tauri event | OS file system events flow into the watcher and out as Tauri events to the webview. Path strings are user-influenced (filenames, project location). |
| Tauri event → JS readFile | The editor-tab listener uses the event payload (`changedPath`) only as an EQUALITY CHECK against the tab's pre-known `filePath`. It does NOT pass the payload back into `readFile` directly — it calls `readFile(filePath)` (the closure-captured trusted path). This is intentional: prevents an event-payload-injection vector. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01-01 | Tampering | file_watcher.rs filter | mitigate | Hard-coded ignore list (`.git/`, `.planning/`, `node_modules/`, `target/`, `.DS_Store`) using `path.contains` checks; cannot be bypassed via crafted filenames since match is on substring of `to_string_lossy()`. |
| T-21-01-02 | DoS | file_watcher.rs (event flood) | mitigate | 300ms debounce via notify_debouncer_mini; emits at most one event per debounce window (first surviving path). A misbehaving editor cannot cause >~3 emits/second. |
| T-21-01-03 | Information disclosure | event payload (changed path) | accept | Path is the file the user just modified in their own editor — no new disclosure beyond what the file tree already shows. |
| T-21-01-04 | Tampering (cross-file content swap) | editor-tab listener | mitigate | Listener calls `readFile(filePath)` using the closure-captured `filePath`, NOT `event.payload`. Payload is used only for equality matching to gate the reload. Even if the watcher is compromised, it cannot make the editor read a different file. |
</threat_model>
