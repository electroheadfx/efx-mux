**3. Create `src-tauri/src/server.rs`:**

Create a new Rust module with these components:

**ServerProcess managed state:**
```rust
pub struct ServerProcess(pub Mutex<Option<std::process::Child>>);
```

**`start_server` command (per D-06, D-07):**
- Accepts `cmd: String`, `cwd: String` via Tauri command
- First calls `stop_server_inner()` to kill any existing process
- Validates `cwd` exists and is a directory (return error if not)
- Spawns `Command::new("sh").args(["-c", &cmd])` with:
  - `.current_dir(&cwd)`
  - `.stdout(Stdio::piped())`
  - `.stderr(Stdio::piped())`
  - `.process_group(0)` (requires `use std::os::unix::process::CommandExt;`)
- Takes `child.stdout` and `child.stderr`, spawns reader threads for each
- Reader threads: read into 4096-byte buffer, convert via `String::from_utf8_lossy`, emit via `app.emit("server-output", &text)` (per D-06)
- Stores child in `ServerProcess` managed state
- Returns `Ok(())`

**`stop_server` command (per D-07):**
- Locks `ServerProcess` mutex, takes the `Option<Child>`
- If Some(child): call `unsafe { libc::killpg(child.id() as i32, libc::SIGTERM) }`
- Spawn a thread that waits 3 seconds, then calls `libc::killpg(pid, libc::SIGKILL)` if process still alive (check via `child.try_wait()`)
- Emits `app.emit("server-stopped", &())` after kill
- If None: return Ok(()) silently

**`restart_server` command (per D-07):**
- Calls `stop_server_inner()` then `start_server` logic
- Emits `app.emit("server-output", "[server] --- Restarting ---\n")` between stop and start

**`detect_agent` command (per D-10):**
- Accepts `agent: String`
- If agent is empty or "bash", return `Ok("bash".to_string())`
- Run `Command::new("which").arg(&agent).output()`
- If success: return `Ok(agent)`
- If failure: return `Err(format!("Binary '{}' not found in PATH", agent))`

**Helper `stop_server_inner`:**
Private function that takes `&ServerProcess` and `&AppHandle`, performs the kill logic without being a Tauri command.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user config -> Rust Command | server_cmd from state.json is executed via `sh -c` |
| server stdout -> frontend HTML | ANSI output rendered as HTML in log viewer |
| user config -> URL open | server_url opened in system browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | server.rs start_server | accept | server_cmd comes from user's own state.json config file. User configures their own commands. No external/untrusted input. |
| T-07-02 | Tampering | server.rs start_server cwd | mitigate | Validate cwd exists and is a directory before `Command::current_dir()`. Return error if invalid. |
| T-07-03 | Spoofing | ansi-html.ts | mitigate | HTML-escape all server output BEFORE processing ANSI codes. Prevents XSS via crafted server output. Use `&amp;`, `&lt;`, `&gt;` escaping. |
| T-07-04 | Information Disclosure | server-bridge.ts openUrl | accept | URL comes from user's own config or parsed from their own server output. No sensitive data exposure risk. |
| T-07-05 | Denial of Service | server.rs process zombie | mitigate | Kill server process group on window close (CloseRequested handler). Use SIGTERM + 3s timeout + SIGKILL fallback. |
</threat_model>
