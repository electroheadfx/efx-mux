# Quick 260416-l12: Lock Minimap Drag to Vertical Only Summary

Anti-drag CSS rules on three minimap selectors (overlay-container, overlay, canvas) preventing native browser drag from hijacking horizontal mouse movement.

## What Changed

Added `user-select: none` and `-webkit-user-drag: none` to three minimap selectors in the global CSS overrides section. The canvas element also received `pointer-events: none` to ensure mouse events pass through to the overlay handler above it.

### Rules Added

| Selector | Properties Added |
|----------|-----------------|
| `.cm-minimap-overlay-container` | `user-select: none`, `-webkit-user-drag: none` (new rule) |
| `.cm-minimap-inner > canvas` | `user-select: none`, `-webkit-user-drag: none`, `pointer-events: none` |
| `.cm-minimap-overlay` | `user-select: none`, `-webkit-user-drag: none` |
