### Gaps Summary

**Root cause:** The plan ran `npm install` to add new dependencies, but this project uses pnpm as its package manager (evidenced by `pnpm-lock.yaml` and the `.modules.yaml` listing `packageManager: pnpm@10.28.0`). npm install correctly wrote to `package-lock.json` and to the npm cache, but did NOT install packages into `node_modules` via pnpm. As a result:

- `vite`, `preact`, `@preact/signals`, `tailwindcss`, `@tailwindcss/vite`, `@preact/preset-vite`, `typescript`, `vitest`, `jsdom`, `marked` are declared in `package.json` but absent from `node_modules`
- `npm run dev` fails: "sh: vite: command not found"
- `npm run build` fails: "sh: tsc: command not found" (then "sh: vite: command not found")
- `npm run test` fails: "sh: vitest: command not found"

**Fix:** Run `pnpm install` from the project root. This will read `package.json`, resolve all dependencies into pnpm's content-addressable store, update `pnpm-lock.yaml`, and symlink binaries into `node_modules/.bin`. After this single command, all three failing truths should pass.

**All source code artifacts are correct and complete.** The migration was done properly at the code level — TypeScript, JSX/TSX, signals, Tauri API imports, cleanup of old Arrow.js files. The only gap is that the new package manager dependencies were not installed via pnpm.
