## Spacing Scale

This project uses a compact desktop-native scale defined in `src/tokens.ts`. All values are multiples of 1px at the micro end; tab-bar anatomy is the binding constraint.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 1px | Sub-pixel separators |
| sm | 2px | Tight intra-element gap |
| md | 4px | Icon gap inside tab label, inner padding |
| lg | 6px | Tab bar vertical padding (py) |
| xl | 8px | Tab horizontal padding (px) — established by UnifiedTabBar |
| 2xl | 10px | — |
| 3xl | 12px | Tab horizontal padding (px) — established by UnifiedTabBar |
| 4xl | 16px | Content area padding |
| 5xl | 20px | — |
| 6xl | 28px | — |

Source: `src/tokens.ts` spacing object — pre-populated.

**Exceptions:**
- Tab close `×` button hit target: 20×20px (WKWebView safe minimum, no change from Phase 17).
- Sticky tab visual treatment: same padding as dynamic tabs (`8px 12px`), no extra inset.
- Optional divider between sticky segment and dynamic segment: 1px vertical rule at `colors.bgBorder` (#243352), 12px tall, centered in tab bar.

**Accent reserved for:**

3. `+` dropdown trigger hover state (border color)
4. Vertical split handle hover (`split-handle-v:hover` — existing, not changed by this phase)
5. Sticky tab icon color when active (same treatment as terminal icon when active)

**Tab color states (binds to 11px label):**
- Active: `colors.textPrimary` (#E6EDF3), weight 600, `2px solid accent` bottom border

**Sticky-tab visual distinction:** sticky tabs (File Tree, GSD) render with the same color rules as dynamic tabs. No background fill or locked-state tinting. The only visible sticky distinction is the absence of a `×` close button.

## Component Inventory

New and modified components for this phase. Executor must not create net-new components when an existing one satisfies the contract.

### Modified: `UnifiedTabBar` (`src/components/unified-tab-bar.tsx`)

**New props:**
- `scope: 'main' | 'right'` — governs which signals/state registry the bar operates over
- When `scope === 'right'`: plus-menu items = Terminal (Zsh), Agent, Git Changes (no File / Editor items)
- When `scope === 'right'`: sticky tabs rendered first: `file-tree` then `gsd`, in that fixed order

**Sticky tab rendering contract:**
- Tab type `'file-tree'`: icon = `FolderOpen` (size 14, `colors.textMuted` when inactive, `colors.accent` when active); label = "File Tree"; no `×` button; no drag handle (pinned position 0)
- Tab type `'gsd'`: icon = `ListChecks` (size 14, same color rules); label = "GSD"; no `×` button; no drag handle (pinned position 1)
- A 1px vertical divider (`colors.bgBorder`, h-3, self-centered) renders between the sticky pair and the dynamic-tabs segment when at least one dynamic tab exists
- Drag reorder: dynamic tabs can only reorder among themselves (indices >= 2 in right scope); drag-over onto a sticky tab position is rejected (no drop highlight, no reorder)

**Plus-menu items (scope `'right'` only):**

| Label | Icon | Action |
|-------|------|--------|
| Terminal (Zsh) | `Terminal` size 14 | `createNewTab({ scope: 'right' })` |
| Agent | `Bot` size 14 | `createNewTab({ scope: 'right', isAgent: true })` |
| Git Changes | `FileDiff` size 14 | move/open gitChangesTab in right scope |

**Git Changes ownership handoff visual:** when Git Changes tab moves from main to right panel (D-07), no animation — tab disappears from main tab bar and appears in right tab bar immediately. Active tab in main panel falls back to the previously active tab or the first available tab.

### Modified: `terminal-tabs.tsx` (`src/components/terminal-tabs.tsx`)

No visual changes. Scope parametrization is structural only.

### Modified: `right-panel.tsx` (`src/components/right-panel.tsx`)

**New layout contract:**
- Remove `.right-top`, `.right-bottom`, `.split-handle-h[data-handle="right-h"]` DOM nodes
- Render a single full-height flex-col inside `.right-panel`
- Top: `UnifiedTabBar scope="right"` — same 33px height as main panel tab bar (32px content + 1px bottom border)
- Below: flex-1 content area with `display: none / block` switching per active tab:
  - `file-tree` → `<FileTree />`
  - `gsd` → `<GSDPane />`
  - `git-changes` → `<GitChangesTab />`
  - terminal/agent tab → `div.terminal-containers[data-scope="right"]` wrapper (always mounted, WebGL context preserved)

### Reused as-is (no visual changes)

- `CrashOverlay` / `ActiveTabCrashOverlay` — existing crash pill style
- `ConfirmModal` — existing three-button modal (agent-quit pattern from Phase 17)
- `Dropdown` (`dropdown-menu.tsx`) — existing plus-menu dropdown style

## Interaction Contract

### Tab switching
- Single click on any tab (sticky or dynamic) → activates that tab's content pane
- Keyboard: existing `Tab` / `Shift+Tab` within tab bar (role=tab, aria-selected) — no new shortcut for this phase

### Rename
- Double-click on a dynamic tab label → enters rename mode (inline input, 100px wide)
- Sticky tabs (File Tree, GSD) are not renameable — double-click on label does nothing
- Confirm rename: Enter key or blur
- Cancel rename: Escape key

### Close
- `×` button appears on hover for dynamic tabs (terminal, agent, git-changes) only
- Sticky tabs have no `×` — this is enforced by not rendering the close button element
- Agent tab close → opens `ConfirmModal` ("Quit Agent Only" / "Close Terminal" / "Cancel")
- Terminal tab close → immediate close (no confirmation), PTY destroyed

### Drag reorder
- Dynamic tabs only: mousedown → mousemove threshold 5px → drag starts (existing WKWebView-safe pattern)
- Drag target: can reorder among dynamic tabs (positions 2..N in right scope)
- Drop onto sticky tab position: no-op, drag cancelled
- Visual during drag: dragging tab renders with `opacity: 0.5`; drop indicator is a 2px accent-colored vertical rule between potential positions (existing pattern from Phase 17)

### Wheel scroll
- Horizontal wheel scroll on the right-panel tab bar scrolls the overflow tab list (matching main panel behavior)

### Plus menu (`+` dropdown)
- Click `+` button → opens Dropdown with 3 items: Terminal (Zsh), Agent, Git Changes
- `+` button style: same `titlebar-add-btn` visual or inline-rendered `+` text button at `colors.textMuted`, hover accent — match existing main-panel `+` button style exactly
- If Git Changes already open in main panel: clicking Git Changes in right-panel `+` menu moves the tab (not duplicate)
- If Git Changes already open in right panel: Git Changes item is disabled (greyed out at 40% opacity) in the dropdown

### Crash overlay
- When a right-panel PTY exits, the `CrashOverlay` renders over that terminal tab's content (existing component, no style change)
- "Restart" button re-spawns the PTY with the same `-r<N>` session name

### State persistence UI
- No user-visible persistence UI — save/restore is silent
- On restore, if persisted active tab ID not found → silently default to "File Tree" sticky tab (no toast, no prompt)
