<objective>
Close two related Phase 18 UAT bugs with a single shared root cause:

**UAT Test 16 (major):** Drop a Finder file ON a folder row — file lands in WRONG folder. Tauri 2.10.x ships a documented bug (Issue #10744): on macOS with `titleBarStyle: "Overlay"`, `onDragDropEvent.payload.position.y` is offset by ~28 CSS px (the macOS title-bar height) from `MouseEvent.clientY` / `Element.getBoundingClientRect().top`. main.tsx already corrects DPR but does not subtract the offset. Result: a drop at viewport clientY=Y is reported as Y+28, so the y-only hit-test in file-tree.tsx matches a row 1-2 positions off.

**UAT Test 17 (BLOCKER):** Drop a Finder file OUTSIDE the file-tree container (e.g. on the terminal panel) — file is silently copied into a "random" folder. Same root cause + a second bug: the row hit-test checks ONLY y-bounds. Because rows tile the entire vertical span, ANY cursor y aligned with any row matches a row even when cursor x is far outside the file-tree panel. The "Drop target outside file tree" toast guard exists but is structurally unreachable.

Both bugs share the fix surface (main.tsx coordinate translation + file-tree.tsx hit-tests) so they MUST be fixed together to avoid regression cycles.

Purpose: Restore correct drag-drop targeting (UAT Test 16) AND restore the data-integrity guard for outside-container drops (UAT Test 17). The Test 17 fix is a BLOCKER because silent copy of files to wrong locations is a data-integrity issue.
Output: One constant + one subtraction in main.tsx. Two-line bounds-check addition to four call sites in file-tree.tsx. Three to four regression tests that mock getBoundingClientRect with realistic non-zero rects so the production geometry is actually exercised in CI. Plus one 1-character update to an existing test (file-tree.test.tsx line ~480) so its zero-rect-jsdom assertion still fires after the x-axis bounds check is added.
</objective>

    await getCurrentWebviewWindow().onDragDropEvent((event) => {
      const payload = event.payload as
        | { type: 'enter'; paths: string[]; position: { x: number; y: number } }
        | { type: 'over'; paths: string[]; position: { x: number; y: number } }
        | { type: 'drop'; paths: string[]; position: { x: number; y: number } }
        | { type: 'leave' };
      if (payload.type === 'leave') {
        document.dispatchEvent(new CustomEvent('tree-finder-dragleave'));
        return;
      }
      const projectName = activeProjectName.value;
      const project = projectName ? projects.value.find(p => p.name === projectName) : undefined;
      const projectPath = project?.path ?? '';
      const paths = 'paths' in payload ? payload.paths : [];
      const anyOutside = paths.length > 0 && paths.some(p => !projectPath || !p.startsWith(projectPath));
      if (!anyOutside) return;
      // Convert physical-pixel position to CSS pixels (per RESEARCH.md §1).
      const dpr = window.devicePixelRatio || 1;
      const detail = {
        paths,
        position: { x: payload.position.x / dpr, y: payload.position.y / dpr },
        // ↑↑↑ Y is in WINDOW coordinates (origin = top of macOS title bar)
        // ↓↓↓ getBoundingClientRect uses VIEWPORT coordinates (origin = top of webview content)
        // The 28px overlay title-bar height MUST be subtracted from y here.
      };
      if (payload.type === 'enter' || payload.type === 'over') {
        document.dispatchEvent(new CustomEvent('tree-finder-dragover', { detail }));
      } else if (payload.type === 'drop') {
        document.dispatchEvent(new CustomEvent('tree-finder-drop', { detail }));
      }
    });
  } catch (err) {
    console.warn('[efxmux] onDragDropEvent subscribe failed:', err);
