# Phase 18 — UI Design Contract

> Visual and interaction contract for file tree enhancements: per-row context menu with submenu, delete flow with confirmation, intra-tree drag-to-move, Finder drop import, inline create file/folder, and external-editor launch. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase 18-specific exceptions:**
- **Submenu horizontal offset:** `2px` from parent menu's right edge (or left edge when flipped). Matches macOS native submenu gap.
- **Drop-indicator border width:** `2px` solid accent (matches Phase 17 unified-tab-bar drop marker `2px solid ${colors.accent}`).
- **Inline input row height:** matches tree row height (`fileTreeLineHeight.value * 2 + icon height ≈ 24px` effective row). No new dimension.
- **Submenu chevron size:** `10px` (existing `ChevronIcon` size in `file-tree.tsx`).
- **Drag ghost offset:** cursor-relative `-40px` left (matches `unified-tab-bar.tsx:635` pattern).
- **Finder drop-zone outer glow:** `1px` inset outline in accent, `8px` outer soft shadow with accent at 20% opacity.
- **Submenu hover delay:** `150ms` (RESEARCH.md §6; matches macOS native behavior).

## Visual Hierarchy

**Primary focal point:** The file tree list itself. Header buttons are secondary chrome. Context menus and modals are transient overlays — they elevate above the tree via `zIndex: 1000` (menu) and `zIndex: 50` (modal).

**Affordance precedence (most to least prominent):**
1. Selected tree row (elevated bg)
2. Drop-target indicator (2px accent left border + tint)
3. Context menu item hover (bg shift)
4. Header button hover (bg shift)
5. Default tree row (no decoration)

## Interaction Contracts

### Delete Flow (TREE-01, TREE-02)

| Trigger | Behavior |
|---------|----------|
| Right-click tree row → "Delete" menu item | Selection snaps to clicked row. Menu closes. For folders, `count_children(path)` invoked in parallel to decide message. `showConfirmModal(...)` opens with file or folder copy per table above. |
| Delete key (or Cmd+Backspace) when tree scroll container has focus and a row is selected | Same as context menu "Delete" — `count_children` + `showConfirmModal`. The keydown handler attaches only when the scroll container has focus (existing `tabIndex=0`). No global listener. |
| User clicks "Delete" in modal | `await deleteFile(path)` via `file-service.ts`. On success: hide modal, emit `git-status-changed` (Tauri event), tree re-renders without row. On failure: hide modal, show error toast `"Could not delete {name}"` with hint. |
| User clicks "Cancel" / Escape / backdrop | Hide modal. No FS mutation. |
| `selectedIndex` after delete | Clamp to `min(prevIndex, flatList.length - 1)`. If tree is empty → `selectedIndex = 0`. |

### Create Flow (MAIN-03)

| Trigger | Behavior |
|---------|----------|
| Right-click folder → "New File" / "New Folder" | Folder auto-expands if collapsed. InlineCreateRow inserted as first child (depth = folder.depth + 1). Input autofocused. |
| Header `[+]` → "New File" / "New Folder" | Target resolved per D-23 (selected folder / file's parent / project root). If target is a collapsed folder, expand it. InlineCreateRow inserted. |
| User types valid name + Enter | `createFile(target)` or `createFolder(target)` invoked. On success: row replaced by actual tree node, `selectedIndex` points at new node, emit `git-status-changed`. On conflict/invalid path (Rust returns error) → error text appears below input, input stays focused, row stays mounted. |
| User presses Escape | Row immediately unmounted. No FS call. `selectedIndex` restored to prior selection. |
| User clicks outside (blur) | Same as Enter — attempt commit. On error, row stays mounted (re-focus input). |
| Empty name on commit | Inline error `"Name required"`. Row stays. |
| Name contains `/` or `\0` | Inline error `"Invalid characters"`. Row stays. |
| Target already exists | Inline error `"{name} already exists"`. Row stays. |

### External Editor Launch (TREE-03)

| Trigger | Behavior |
|---------|----------|
| Right-click file or folder → "Open In" → editor | `invoke('launch_external_editor', { app: 'Zed'|'Visual Studio Code'|..., path })` (Rust: `open -a {app} {path}`). Menu closes immediately. |
| Right-click → "Open with default app" | `invoke('open_default', { path })` (Rust: `open {path}`). |
| Right-click → "Reveal in Finder" | `invoke('reveal_in_finder', { path })` (Rust: `open -R {path}`). |
| Header "Open In" button | Same submenu as row-level "Open In" but targets active project root (`activeProject.path`). Submenu anchored below the header button (downward) instead of beside (rightward). |
| Editor detection fails at runtime (app not installed despite `which` hit) | Rust command returns error. Frontend shows error toast `"Could not launch {app}"` with no hint. |
| No editors detected at app start | "Open In" hidden entirely (both row menu and header button). Header button simply not rendered. |

### Intra-Tree Drag-to-Move (TREE-04)

Follow `unified-tab-bar.tsx:573-740` pattern verbatim. WKWebView constraint applies per D-11.

| Step | Behavior |
|------|----------|
| `mousedown` on tree row (left button, not on a button/input) | Store `sourceRow`, `startX`, `startY`. Attach document-level `mousemove`/`mouseup`. No visual change yet. |
| `mousemove` before threshold | If `sqrt(dx² + dy²) < 5px`, do nothing. |
| `mousemove` crosses 5px threshold | Set source row opacity to `0.4`. Create ghost clone appended to body. Ghost follows cursor. |
| `mousemove` after threshold | Update ghost position. Find tree row under cursor via `document.elementFromPoint()`. If hover row is a folder (or the empty tree area below rows), apply `borderLeft: 2px solid accent` + `backgroundColor: accentMuted`. Clear indicators from all other rows. |
| `mouseup` | Resolve drop target per D-12: drop on folder → move into folder; drop on file → move into file's parent folder; drop on empty area → move to project root. `await renameFile(sourcePath, targetPath)`. Emit `git-status-changed`. |
| Conflict at target (D-20) | Rust returns error. Show toast `"File exists: {name}"`. No FS mutation. Source row returns to full opacity. |
| Drop on own parent (no-op) | Abort silently. No IPC call. No error. |
| Drop on self (folder onto itself or descendant) | Abort silently. No IPC call. |
| `mouseup` without crossing threshold | Treat as a click (falls through to existing click handler — opens file or toggles folder). |
| Cleanup | Restore source opacity. Remove ghost node. Clear all border-left / bg-tint styles. Detach document listeners. |

### Finder Drop Import (TREE-05)

Tauri 2 `onDragDropEvent` registered once in `main.tsx` (or dedicated module) and routed to the file tree via `CustomEvent`.

| Payload type | Behavior |
|--------------|----------|
| `enter` | Check if any path in `event.payload.paths` is inside active project root. If ALL paths are inside → ignore (intra-drag owns this). If ANY path is outside → activate FinderDropOverlay on tree container. |
| `over` | Update target-row highlight based on cursor `position` (convert from physical px via `window.devicePixelRatio`). Same highlight rules as intra-drag (folder → folder, file → parent folder, empty → project root). |
| `drop` | Deactivate overlay. For each external path: `await copyPath(sourcePath, targetPath)`. On conflict (D-20) show toast `"File exists: {name}"`. Emit `git-status-changed` after each success. Emit once more at end to cover batch. |
| `leave` | Deactivate overlay. Clear all row highlights. No FS mutation. |
| Cursor outside tree container | Overlay stays but target-row highlight clears. |
| Drop on non-tree area of window | `drop` still fires but target-row resolution returns null → toast `"Drop target outside file tree"` (no FS call). |

### Submenu Interaction

| Trigger | Behavior |
|---------|----------|
| Mouse enters parent row with `children` | Start 150ms timer. If mouse stays, open submenu at `parentRect.right + 2`. |
| Mouse leaves parent row | Start 150ms close timer. If mouse enters submenu within 150ms, cancel timer; submenu stays. |
| Mouse leaves submenu (without re-entering parent) | Close submenu immediately; keep parent menu open. |
| Click on submenu item | Invoke action, close entire menu stack. |
| Escape key | Close both submenu and parent menu. |
| Click outside entire menu stack | Close both. |
| Viewport flip (right edge) | Submenu renders at `parentRect.left - submenuWidth - 2`. |
| Viewport flip (bottom edge) | Submenu aligns `top = parentRect.bottom - submenuHeight`. |

### Tree Refresh After Ops

Per CONTEXT.md "Tree refresh mechanism" (Claude's discretion):

**Decision:** Reuse `git-status-changed` Tauri event (already emitted by `writeFile` and consumed by the Git sidebar). Rust commands `delete_file`, `rename_file`, `create_file`, `create_folder`, `copy_path` MUST `emit('git-status-changed', ())` after successful completion. File tree adds a listener that calls `initTree()` (tree mode) or `loadDir(currentPath)` (flat mode). This avoids adding a second event name and keeps the sidebar and tree coherent.
