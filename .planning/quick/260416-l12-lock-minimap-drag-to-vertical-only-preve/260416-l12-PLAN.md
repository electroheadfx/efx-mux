<objective>
Lock minimap drag to vertical only by preventing the browser's native drag-and-drop from hijacking horizontal mouse movement on the minimap overlay.

Purpose: The `@replit/codemirror-minimap` overlay handler only tracks `clientY`, but its `onMouseDown` does not call `preventDefault()`. When the user drags horizontally, the browser's native drag mechanism activates and visually pulls the overlay element off the minimap. Adding `user-select: none` and `-webkit-user-drag: none` on the overlay container, overlay, and canvas elements suppresses this.

Output: Updated `src/styles/app.css` with anti-drag rules in the existing minimap overrides section.
</objective>
