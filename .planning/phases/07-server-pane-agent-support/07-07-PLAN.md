**In src/styles/app.css:**

1. In `.server-pane-logs` (around line 173), change `color: inherit` to `color: var(--color-text)`. This sets the default text color explicitly but does NOT use `inherit` which would override inline `style="color:..."` on child spans. The `color` CSS property on a parent does cascade to children, but inline styles have higher specificity. The issue is that `inherit` is being used on the container itself and Tailwind 4 layers may cause specificity conflicts. Using the explicit var ensures the container has a base color while inline spans from ansiToHtml() override it correctly.

   Actually, the real fix: `color: inherit` on `.server-pane-logs` should NOT override inline styles on child spans -- inline styles always win specificity. The actual problem may be that the `<div dangerouslySetInnerHTML>` wrapper or Tailwind's text color classes are interfering. Let me check: the parent `<div class="server-pane server-pane-logs">` inherits from `.server-pane` which inherits from the app root. The spans inside have `style="color:#dc322f"` which is inline and should win.

   The REAL fix: Change `color: inherit` to `color: var(--color-text)` on `.server-pane-logs`. This provides a default text color for non-ANSI text. The inline `style="color:..."` on ansiToHtml spans will override this correctly because inline styles beat CSS rules regardless of specificity. The `inherit` keyword was likely causing issues because the inherited color from a Tailwind utility class on a parent element was overriding everything.

**In src/components/server-pane.tsx:**

2. Fix the auto-scroll useEffect to use `requestAnimationFrame` so the browser has painted the new content before measuring scroll position:
   ```typescript
   useEffect(() => {
     const el = logRef.current;
     if (!el) return;
     requestAnimationFrame(() => {
       if (!el) return;
       // Always auto-scroll to bottom when new logs arrive
       // User can still scroll up to review, but new output snaps to bottom
       el.scrollTop = el.scrollHeight;
     });
   }, [serverLogs.value]);
   ```
   
   Note: The UAT diagnosis says to auto-scroll. The simplest correct behavior is to always scroll to bottom on new output. If the user explicitly scrolled up, new output still scrolls down -- this matches terminal behavior and is what the user requested ("I want an auto scroll"). Remove the `isNearBottom` check to make auto-scroll unconditional.
