       ```rust
       /// Phase 20 D-19: Kill legacy "<project>-right" tmux sessions from the pre-Phase-20
       /// right-panel layout. Best-effort; silently ignores missing sessions. Called once
       /// at app bootstrap after `cleanup_dead_sessions`.
       ///
       /// Returns the list of session names actually killed (for logging). Never returns Err —
       /// migration is best-effort and should not block startup.
       #[tauri::command]
       pub fn kill_legacy_right_sessions(project_names: Vec<String>) -> Result<Vec<String>, String> {
           let mut killed = Vec::new();
           for name in project_names {
               // Mirror pty.rs session-name sanitizer: alphanumeric + '-' + '_', lowercased.
               let sanitized: String = name
                   .chars()
                   .filter(|c| c.is_alphanumeric() || *c == '-' || *c == '_')
                   .collect::<String>()
                   .to_lowercase();
               if sanitized.is_empty() {
                   continue;
               }
               let target = format!("{}-right", sanitized);
               // tmux returns non-zero when session doesn't exist — treat as no-op.
               if let Ok(out) = std::process::Command::new("tmux")
                   .args(["kill-session", "-t", &target])
                   .output()
               {
                   if out.status.success() {
                       killed.push(target);
                   }
               }
           }
           Ok(killed)
       }
       ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → Rust command | `kill_legacy_right_sessions(project_names: Vec<String>)` receives project names from frontend |
| Disk (`state.json`) → Frontend | `loadAppState` reads a potentially-tampered older `state.json` with legacy keys |
| Rust → tmux binary | `tmux kill-session -t <target>` shell-out with computed target string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-07 | Tampering / Elevation | `kill_legacy_right_sessions` project name → tmux target | mitigate | Sanitize each project name via `is_alphanumeric() \|\| '-' \|\| '_'` filter + `.to_lowercase()` before `format!("{}-right", ...)`. Mirrors existing pty.rs sanitizer (`src-tauri/src/terminal/pty.rs:62-65`). Empty-string skip prevents `format!("-right")` from targeting an arbitrary session. |
| T-20-08 | Tampering | Legacy keys in older `state.json` | mitigate | Keys are deleted (not trusted, not propagated) during `loadAppState`. Silent + idempotent per D-20. |
| T-20-09 | Denial of Service | `kill_legacy_right_sessions` loop blocking startup | mitigate | Per-project tmux invocation is serial but bounded by project count (typical < 20). Inner `Command` uses `.output()` which returns on command exit; no indefinite wait. Bootstrap caller wraps in `.catch()` (Plan 04) so failure never blocks UI. |
| T-20-10 | Information Disclosure | `killed: Vec<String>` returned to frontend logs | accept | Killed names are `<project>-right` — echoes user-controlled project names the user already sees in their sidebar. No new disclosure surface. |
