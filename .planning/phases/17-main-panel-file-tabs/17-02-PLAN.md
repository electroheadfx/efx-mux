Define the unified tab type system (per RESEARCH.md Pattern 4):

```typescript
// Tab type discriminated union
interface BaseTab {
  id: string;
  type: 'terminal' | 'editor' | 'git-changes';
}

interface EditorTabData extends BaseTab {
  type: 'editor';
  filePath: string;
  fileName: string;
  dirty: boolean;
}

interface GitChangesTabData extends BaseTab {
  type: 'git-changes';
}

// UnifiedTab wraps terminal tabs or adds editor/git-changes tabs
type UnifiedTab = { type: 'terminal'; terminalTabId: string } | EditorTabData | GitChangesTabData;
```

NOTE: Terminal tabs are NOT duplicated. UnifiedTab entries of type 'terminal' hold a reference to the terminal tab's `id` in `terminalTabs` signal. This avoids copying Terminal/FitAddon objects. The unified tab list is derived: terminal entries point into the existing terminalTabs signal.

Module-level signals:
```typescript
export const editorTabs = signal<EditorTabData[]>([]);
export const gitChangesTab = signal<GitChangesTabData | null>(null);
export const activeUnifiedTabId = signal<string>('');
```

The combined tab list is derived by computing: `[...terminalTabs mapped to unified, ...editorTabs, gitChangesTab if present]`. Use a `computed` signal for this:
```typescript
import { computed } from '@preact/signals';
export const allTabs = computed<UnifiedTab[]>(() => {
  const terminals: UnifiedTab[] = terminalTabs.value.map(t => ({ type: 'terminal' as const, id: t.id, terminalTabId: t.id }));
  const editors: UnifiedTab[] = editorTabs.value;
  const git: UnifiedTab[] = gitChangesTab.value ? [gitChangesTab.value] : [];
  return [...terminals, ...editors, ...git];
});
```

```typescript
export const tabOrder = signal<string[]>([]);  // ordered list of tab IDs
```

```typescript
  const setupRef = useRef<EditorSetupResult | null>(null);
```

**useEffect keyed on `filePath`:**
1. If `containerRef.current` is null, return
2. Detect language: `const lang = getLanguageExtension(fileName);`
3. Create state: `const setup = createEditorState(content, { language: lang, onSave: handleSave, onDirtyChange: handleDirtyChange });`
4. Create view: `const view = new EditorView({ state: setup.state, parent: containerRef.current });`
5. Store refs: `viewRef.current = view; setupRef.current = setup;`
6. Cleanup: `return () => { view.destroy(); viewRef.current = null; setupRef.current = null; };`

**handleSave function:**
```typescript
async function handleSave(docContent: string) {
  try {
    await writeFile(filePath, docContent);
    setupRef.current?.setSavedContent(docContent);
    setEditorDirty(tabId, false);
    showToast({ type: 'success', message: `Saved ${fileName}` });
  } catch (err) {
    showToast({ type: 'error', message: `Save failed: ${String(err)}` });
  }
}
```

**handleDirtyChange function:**
```typescript
function handleDirtyChange(dirty: boolean) {
  setEditorDirty(tabId, dirty);
}
```
