<task type="auto">
  <name>Task 2: Fix Bash terminal container discovery (Arrow.js ref bug)</name>
  <files>src/components/right-panel.js</files>
  <action>
**Read `.arrow-js/skill/api.md` first.**

Arrow.js does NOT support React-style ref callbacks. The `ref="${(el) => {...}}"` on line 119 is serialized as a string attribute and never executed.

Fix: Replace the ref callback with a post-render querySelector approach.

1. Give the bash terminal container a unique ID: change `class="bash-terminal"` to also include `id="bash-terminal-container"`.

2. Remove the `ref="${(el) => {...}}"` attribute entirely from the div.

3. After the html template is returned, use a `setTimeout` (or `requestAnimationFrame`) to find the container via querySelector and connect. The cleanest pattern for Arrow.js:

```javascript
// After the return html`...` block, schedule the bash terminal connection.
// Use setTimeout(0) to run after Arrow.js has rendered the template to DOM.
setTimeout(() => {
  if (bashTerminalConnected) return;
  bashContainerEl = document.getElementById('bash-terminal-container');
  if (bashContainerEl && state.rightBottomTab === 'Bash') {
    setTimeout(() => connectBashTerminal(), 200);
  }
}, 0);
```

However, since RightPanel returns an html template (Arrow.js component pattern), the setTimeout must be placed BEFORE the return statement but scheduled to run asynchronously. The correct approach:

In the RightPanel function body, AFTER defining `connectBashTerminal` and BEFORE the `return html` statement, add:

```javascript
// Schedule bash terminal discovery after Arrow.js renders the DOM
setTimeout(() => {
  bashContainerEl = document.getElementById('bash-terminal-container');
  if (bashContainerEl && !bashTerminalConnected) {
    setTimeout(() => connectBashTerminal(), 200);
  }
}, 0);
```

Then in the template, change the bash-terminal div to:
```
<div id="bash-terminal-container" class="bash-terminal" style="height: 100%;"></div>
```

Remove the entire `ref="${(el) => {...}}"` attribute.

Do NOT use HTML comments in the Arrow.js template -- they crash Arrow.js.
