# Plan 06-07: UAT Gap Closure

**Goal:** Fix 1 bug and 2 UX enhancements from Phase 6 retest UAT.

## Wave 1 (all independent, can execute in parallel)

### Task 1: Fix MD file watcher to use Recursive mode
**Gap:** Test 3 — GSD Viewer does not auto-refresh on external .md edits
**Root cause:** `file_watcher.rs:65` uses `RecursiveMode::NonRecursive`, so only project root .md files are detected. Most .md files live in subdirectories.
**Fix:**
- `src-tauri/src/file_watcher.rs` line 65: change `RecursiveMode::NonRecursive` to `RecursiveMode::Recursive`

### Task 2: Auto-switch to Diff tab on sidebar file click
**Gap:** Test 4 note — clicking a file in GIT CHANGES should auto-open the Diff tab
**Root cause:** `sidebar.js:208` dispatches `open-diff` event but `right-panel.js` doesn't listen for it to switch tabs.
**Fix:**
- `src/components/right-panel.js`: add `document.addEventListener('open-diff', ...)` that sets `state.rightTopTab = 'Diff'`
- Place listener after `loadActiveProject()` call (around line 55)

### Task 3: GIT CHANGES section fills remaining sidebar height
**Gap:** Test 4 note — GIT CHANGES has constrained height with wasted empty space below
**Root cause:** `sidebar.js:443` git files container has `max-height: 120px` hardcoded. The parent GIT CHANGES div (line 394) doesn't flex-grow.
**Fix:**
- `src/components/sidebar.js` line 443: remove `max-height: 120px`, use `flex: 1; overflow-y: auto;` instead
- `src/components/sidebar.js` line 394: add `flex: 1; min-height: 0;` to the GIT CHANGES container div so it grows to fill remaining sidebar space
- The PROJECTS section (line 386) already has `flex: 1` — change it to `flex: 0 1 auto` so it only takes space it needs, letting GIT CHANGES take the rest. Or better: keep PROJECTS as `flex: 1` and make GIT CHANGES also `flex: 1` so they share space.

## Verification

After all tasks:
1. Edit a .md file in a subdirectory externally → GSD Viewer should auto-refresh
2. Click a file in sidebar GIT CHANGES → right panel should auto-switch to Diff tab
3. Sidebar GIT CHANGES section should use available vertical space (no large empty gap below file list)
