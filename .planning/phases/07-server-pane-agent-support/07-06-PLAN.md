**In src-tauri/src/server.rs:**

1. Replace `ServerProcess(pub Mutex<Option<std::process::Child>>)` with:
   ```rust
   pub struct ServerProcesses(pub Mutex<HashMap<String, ServerEntry>>);
   
   pub struct ServerEntry {
       pub child: std::process::Child,
       pub pid: u32,
   }
   ```
   Import `std::collections::HashMap`.

2. Add a `project_id: String` parameter to `start_server`, `stop_server`, and `restart_server` commands. The project_id is the project name (used as HashMap key).

3. In `start_server`: after spawning, store `ServerEntry { child, pid }` in the HashMap under `project_id`. Before spawning, call `stop_server_for_project` to kill any existing server for that project.

4. Fix `stop_server_inner` -> rename to `stop_server_for_project(app: &AppHandle, project_id: &str)`:
   - Look up the project_id in the HashMap
   - If found, get the PID and call `killpg(pid, SIGTERM)`
   - CRITICAL FIX: After calling killpg, call `libc::waitpid(pid, &mut status, 0)` in a background thread to properly reap the process. This prevents zombies.
   - Remove the entry from HashMap AFTER confirming kill (not immediately)
   - Keep the 3-second SIGKILL fallback thread, but have it also waitpid after SIGKILL

5. Add `pub fn kill_all_servers(app: &AppHandle)` that iterates all entries in the HashMap and kills each one (SIGTERM + SIGKILL fallback). Used by close handler.

6. In `stop_server` command: accept `project_id: String` parameter, call `stop_server_for_project(app, &project_id)`.

7. In `restart_server`: accept `project_id: String`, call `stop_server_for_project` then start.

8. The stdout/stderr reader threads must also use project_id in event names OR the frontend filters by active project. Simplest approach: keep the same `server-output` / `server-stopped` event names but include project_id in the payload. Change emit to:
   ```rust
   // For server-output, emit a JSON object
   let payload = serde_json::json!({ "project": project_id_clone, "text": text });
   let _ = app_clone.emit("server-output", payload);
   
   // For server-stopped
   let payload = serde_json::json!({ "project": project_id_clone, "code": exit_code });
   let _ = app_clone.emit("server-stopped", payload);
   ```
   Clone `project_id` into each thread.

**In src-tauri/src/lib.rs:**

1. Change `app.manage(ServerProcess(...))` to `app.manage(ServerProcesses(Mutex::new(HashMap::new())))`. Import HashMap if not already imported.

2. Update close handler to call `kill_all_servers(&app_handle)` instead of the current single-process kill logic. Get the app handle from the window with `window.app_handle()`.

3. Update `use server::{...}` imports to match renamed types.

**In src/components/server-pane.tsx:**

1. Add a per-project cache Map (module-level, NOT a signal since it stores background state):
   ```typescript
   interface ProjectServerState {
     logs: string[];
     status: 'stopped' | 'running' | 'crashed' | 'unconfigured';
     url: string | null;
   }
   const projectServerCache = new Map<string, ProjectServerState>();
   ```

2. Add helper functions:
   ```typescript
   function saveCurrentProjectState(projectName: string) {
     projectServerCache.set(projectName, {
       logs: serverLogs.value,
       status: serverStatus.value,
       url: detectedUrl.value,
     });
   }
   
   function restoreProjectState(projectName: string) {
     const cached = projectServerCache.get(projectName);
     if (cached) {
       serverLogs.value = cached.logs;
       serverStatus.value = cached.status;
       detectedUrl.value = cached.url;
     } else {
       serverLogs.value = [];
       serverStatus.value = 'stopped';
       detectedUrl.value = null;
     }
   }
   ```

3. Export `saveCurrentProjectState` and `restoreProjectState` for use by main.tsx.

4. Update `resetServerPane()` to just call `restoreProjectState` for the new project (or reset to defaults if no cache).

5. Update event listeners to filter by active project:
   - In `listenServerOutput` callback: check if `project === activeProjectName.value`, only update signals if it matches the active project. Always update the cache for the correct project regardless.
   - In `listenServerStopped` callback: same project filtering.

6. Update button handlers to pass `activeProjectName.value` as projectId:

   - `handleRestart`: `await restartServer(proj.server_cmd, proj.path, activeProjectName.value)`
