    ### copy_path (file OR recursive dir)

    ```rust
    /// Recursive copy helper (std only — matches file_ops.rs style).
    fn copy_dir_recursive(src: &Path, dst: &Path) -> std::io::Result<()> {
        std::fs::create_dir_all(dst)?;
        for entry in std::fs::read_dir(src)? {
            let entry = entry?;
            let ty = entry.file_type()?;
            let dst_child = dst.join(entry.file_name());
            if ty.is_dir() {
                copy_dir_recursive(&entry.path(), &dst_child)?;
            } else {
                std::fs::copy(entry.path(), &dst_child)?;
            }
        }
        Ok(())
    }

    /// Synchronous inner implementation of copy_path for testing.
    pub fn copy_path_impl(from: &str, to: &str) -> Result<(), String> {
        if !is_safe_path(from) || !is_safe_path(to) {
            return Err("Invalid path: directory traversal not allowed".to_string());
        }
        let from_p = Path::new(from);
        let to_p = Path::new(to);
        if to_p.exists() {
            return Err(format!("Target exists: {}", to));
        }
        let meta = std::fs::metadata(from_p).map_err(|e| e.to_string())?;
        if meta.is_file() {
            std::fs::copy(from_p, to_p).map(|_| ()).map_err(|e| e.to_string())
        } else if meta.is_dir() {
            copy_dir_recursive(from_p, to_p).map_err(|e| {
                // Best-effort cleanup on partial failure (RESEARCH.md §2 partial-failure policy).
                let _ = std::fs::remove_dir_all(to_p);
                e.to_string()
            })
        } else {
            Err("Unsupported file type".to_string())
        }
    }

    /// Copy a file or directory (Phase 18, D-17).
    #[tauri::command]
    pub async fn copy_path(from: String, to: String, app: AppHandle) -> Result<(), String> {
        if !is_safe_path(&from) || !is_safe_path(&to) {
            return Err("Invalid path: directory traversal not allowed".to_string());
        }
        let from_clone = from.clone();
        let to_clone = to.clone();
        spawn_blocking(move || copy_path_impl(&from_clone, &to_clone))
            .await
            .map_err(|e| e.to_string())??;
        let _ = app.emit("git-status-changed", ());
        Ok(())
    }
    ```

    ```rust
    fn count_children_walk(dir: &Path, c: &mut ChildCount, cap: u32) -> std::io::Result<()> {
        for entry in std::fs::read_dir(dir)? {
            if c.total >= cap {
                c.capped = true;
                return Ok(());
            }
            let e = entry?;
            let ty = e.file_type()?;
            c.total += 1;
            if ty.is_dir() {
                c.folders += 1;
                let _ = count_children_walk(&e.path(), c, cap);
                if c.capped { return Ok(()); }
            } else {
                c.files += 1;
            }
        }
        Ok(())
    }

    /// Synchronous inner implementation of count_children for testing.
    pub fn count_children_impl(path: &str, cap: u32) -> Result<ChildCount, String> {
        if !is_safe_path(path) {
            return Err("Invalid path: directory traversal not allowed".to_string());
        }
        let mut c = ChildCount { files: 0, folders: 0, total: 0, capped: false };
        count_children_walk(Path::new(path), &mut c, cap).map_err(|e| e.to_string())?;
        Ok(c)
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend (WKWebView) → Tauri IPC | All `invoke(...)` calls cross this boundary. User-controlled path strings arrive from JS via `path`, `from`, `to`, `app` parameters. |
| Tauri command → std::fs | `create_folder`, `copy_path`, `count_children` read and mutate the filesystem. |
| Tauri command → std::process::Command | `launch_external_editor`, `open_default`, `reveal_in_finder` spawn external `open` binaries. `detect_editors` spawns `which`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01-01 | Tampering | `create_folder(path)` | mitigate | Reuse `is_safe_path()` guard on both the sync impl and async wrapper (2-layer check — matches existing `create_file` pattern). Rejects any path containing `..` component. |
| T-18-01-02 | Tampering | `copy_path(from, to)` | mitigate | `is_safe_path()` validates both `from` AND `to` on both layers. Target-exists pre-check prevents silent overwrite. On partial copy failure, `remove_dir_all` cleans the partial tree so no half-copied state remains. |
| T-18-01-03 | Tampering | `count_children(path)` | mitigate | `is_safe_path()` guard. Internal recursion is read-only (`read_dir` + `file_type`) — no mutation surface. |
| T-18-01-04 | Information Disclosure | `count_children(path)` | mitigate | Returns only counts (never paths or names). Cannot enumerate filesystem outside provided path. |
| T-18-01-05 | Injection | `launch_external_editor(app, path)` | mitigate | App name and path passed as SEPARATE argv elements via `Command::new("open").args(["-a", &app, &path])` — bypasses shell word-splitting entirely. No `sh -c`, no string interpolation. Even if `app` contains metacharacters (`;`, `|`, `&`), they are treated as literal characters by argv. |
| T-18-01-06 | Privilege Escalation | `launch_external_editor(app, ...)` | accept | User-chosen editor apps (Zed, VSCode, Cursor, etc.) execute with the user's own privileges — same as double-clicking a file in Finder. Acceptable trust boundary: the user's installed GUI applications. `open -a` delegates to LaunchServices which enforces macOS code-signing. |
| T-18-01-07 | Injection | `open_default(path)` / `reveal_in_finder(path)` | mitigate | Path passed as single argv element to `Command::new("open")`. `is_safe_path()` rejects traversal. No shell interpretation. |
| T-18-01-08 | Denial of Service | `count_children` on huge trees | mitigate | Hard cap at 10,000 entries via `count_children_walk` — returns early with `capped: true` flag. Prevents the UI thread from hanging on `node_modules` (~30k files) or `.git` objects (~30k loose). |
| T-18-01-09 | Denial of Service | `copy_path` on symlink cycles | accept | Per RESEARCH.md §2 A4, symlinks are followed (simpler). Cycle detection deferred to Phase 21+ if observed. Risk is LOW for typical project files — no known cycle-prone directories in normal developer workflows. Documented assumption. |
| T-18-01-10 | Tampering | file-service.ts wrappers | mitigate | TypeScript wrappers forward parameters unchanged to `invoke(...)`. All validation happens in Rust (defense-in-depth: the wrapper cannot be trusted because JS runs in a compromised WKWebView). `FileError` class surfaces errors without leaking internal state. |
