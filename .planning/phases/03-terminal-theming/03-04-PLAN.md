  <files>src/theme/theme-manager.js, src/main.js</files>
  <action>
**In src/theme/theme-manager.js:**

1. Add a new exported function `initOsThemeListener()` that:
   - Calls `window.matchMedia('(prefers-color-scheme: dark)')` to get the media query
   - Reads the current OS preference: if OS is dark, set data-theme to 'dark'; if OS is light, set data-theme to 'light'
   - BUT only apply the OS preference if the user has NOT previously set a manual preference. Check: if `localStorage.getItem('efxmux:theme-mode')` is null (first launch), follow OS. If it has a value, honor the stored value (already done in initTheme).
   - Adds a `change` event listener on the media query that calls `setThemeMode(e.matches ? 'dark' : 'light')` when the OS theme changes mid-session
   - This listener should override any manual preference when the OS changes -- the OS is the source of truth for live changes (standard macOS app behavior)

2. Add a helper function `setThemeMode(mode)` (exported) that:
   - Sets `document.documentElement.setAttribute('data-theme', mode)`
   - Sets `localStorage.setItem('efxmux:theme-mode', mode)`
   - This is factored out of toggleThemeMode() to avoid duplication

3. Refactor `toggleThemeMode()` to use `setThemeMode()` internally:
   ```javascript
   export function toggleThemeMode() {
     const current = document.documentElement.getAttribute('data-theme') || 'dark';
     setThemeMode(current === 'dark' ? 'light' : 'dark');
   }
   ```

4. Call `initOsThemeListener()` at the end of `initTheme()`, after the theme is loaded and applied. This ensures OS theme following starts on every launch.

**In src/main.js:**

5. Add a keyboard shortcut for theme toggle. Add to the existing keydown listener block (after the Ctrl+B handler):
   ```javascript
   // Ctrl+Shift+T -- toggle dark/light mode (per THEME-04)
   if (e.ctrlKey && e.shiftKey && e.key === 'T') {
     e.preventDefault();
     toggleThemeMode();
   }
   ```
   This avoids conflicting with Ctrl+T (new tab in Phase 8) by requiring Shift.

6. Ensure `toggleThemeMode` is in the import statement from `./theme/theme-manager.js`. The current import already has `initTheme, registerTerminal` -- add `toggleThemeMode` to that import.

**Do NOT:**
- Add any visible UI button (that's Phase 8 Settings UI)
- Modify theme.css (light mode CSS block already exists and is correct)
- Touch the Rust backend (not needed for this fix)
