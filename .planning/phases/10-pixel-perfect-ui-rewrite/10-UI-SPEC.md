# Phase 10 — UI Design Contract

> Visual and interaction contract for the pixel-perfect UI rewrite. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |

## Visual Focal Point

The primary visual anchor is the **terminal area** in the main panel -- the largest surface area in the layout, rendered in bgDeep (#0B1120). All surrounding chrome (sidebar, tab bars, right panel) is subordinate and uses lighter bgBase (#111927) or bgElevated (#19243A) to draw the eye inward toward the terminal content where the user spends most of their time.

### Collapsed Sidebar Accessibility

When the sidebar is collapsed to the 40px icon strip, each icon button MUST include an `aria-label` attribute describing its action (e.g., `aria-label="Expand sidebar"`, `aria-label="Add project"`, `aria-label="Open preferences"`). This ensures screen reader users can navigate the collapsed state.

## Spacing Scale

Declared values from reference `tokens.ts`. This design uses a **non-standard fine-grained scale** optimized for dense developer tool UI, not the typical 8-point grid.

| Token | Value | Usage |
|-------|-------|-------|
| none | 0px | No spacing |
| xs | 1px | Hairline gaps (label-to-dot) |
| sm | 2px | Tight inline gaps (icon-to-text in badges) |
| md | 4px | Compact padding (badges, status pills, tab gaps) |
| lg | 6px | Section item gaps (project list items, file tree rows) |
| xl | 8px | Standard component padding (agent header, tab bars) |
| 2xl | 10px | Input field padding, card internal margins |
| 3xl | 12px | Card padding (agent header, modals sections) |
| 4xl | 16px | Section breaks, modal body padding |
| 5xl | 20px | Modal outer padding, large section gaps |
| 6xl | 28px | Page-level spacing (sidebar header height) |

Exceptions: Touch targets are not enforced at 44px -- this is a desktop-only macOS app with pointer input. Minimum interactive target size is 24px (tab buttons, sidebar project items).

### Scale Override Justification

This spacing scale contains values that are not multiples of 4 (1px, 2px, 6px, 10px). These are the **verbatim reference values** from the existing design system (`RESEARCH/theme/tokens.ts`) and are locked by user decision D-11 (pixel-perfect design fidelity). Modifying them would violate the phase's core requirement.

Justification for each non-conforming value:

| Value | Use Case | Why Not a Multiple of 4 |
|-------|----------|------------------------|
| 1px | Hairline borders, divider lines, label-to-dot gaps | Sub-pixel precision required for border/separator rendering; 4px would be visually heavy |
| 2px | Active tab bottom indicator, tight icon-to-text gaps in badges | Minimum visible accent indicator; 4px would dominate small badge elements |
| 6px | Project list item vertical gaps, file tree row spacing | Dense list spacing where 4px is too tight and 8px wastes vertical space in a data-heavy sidebar |
| 10px | Input field horizontal padding, card internal margins | Balanced internal padding for form controls; 8px feels cramped, 12px wastes horizontal space |

Note: 12px (4x3) and 28px (4x7) ARE multiples of 4 and conform to the grid.

## Typography

Font family resolution: `'Geist'` for sans, `'GeistMono'` for mono. The `@font-face` declarations in `app.css` register these names (no space in GeistMono). `tokens.ts` must use `mono: 'GeistMono'` to match.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Display | Geist | 20px | 600 | 1.2 | GSD viewer title |
| Subheading | Geist | 15px | 600 | 1.3 | GSD viewer subheadings |
| Project name | Geist | 13px | 500 | 1.3 | Sidebar project names, terminal output, prompt text |
| Body | Geist | 12px | 400 | 1.4 | File tree items, terminal tabs, phase items |
| Tab label | GeistMono | 11px | 500 | 1.2 | Tab labels, server strip text |
| Badge/Status | GeistMono | 10px | 500 | 1.0 | Section labels (uppercase, letter-spacing 1.2px), branch names, badges, status pills |
| Prompt badge | GeistMono | 9px | 500 | 1.0 | Prompt mode badges |

Only 2 weights used per font family:
- **Geist:** 400 (body) + 500-600 (headings/labels)
- **GeistMono:** 500 (all monospace UI elements)

### Scale Override Justification

This typography scale declares 7 font sizes (9, 10, 11, 12, 13, 15, 20px) rather than the typical maximum of 4. These are the **verbatim reference values** from the existing design system (`RESEARCH/theme/tokens.ts` fontSizes map: xs:9, sm:10, md:11, base:12, lg:13, xl:15, 2xl:20) and are locked by user decisions D-11 (pixel-perfect design fidelity) and D-13 (reference font size map).

The 7 sizes map to **4 semantic tiers**, with sub-sizes within tiers for fine-grained density control in a developer tool UI:

| Tier | Sizes | Semantic Role |
|------|-------|---------------|
| **Tier 1 -- Display** | 20px | Headings, titles (used sparingly, one per view) |
| **Tier 2 -- Content** | 15px, 13px | Subheadings (15px) and primary labels like project names (13px) |
| **Tier 3 -- Body** | 12px, 11px | Default body text (12px) and tab/strip labels in monospace (11px) |
| **Tier 4 -- Caption** | 10px, 9px | Status badges and section labels (10px) and prompt mode badges (9px) |

Each tier serves a distinct reading distance and information hierarchy level. The sub-sizes within tiers exist because dense developer tooling requires finer typographic discrimination than consumer UI -- a sidebar project name (13px) must be visually distinct from body file tree text (12px) at a glance.

## Component Visual Contracts

### Sidebar
- Width: CSS variable `--sidebar-w` (drag-resizable, default 280px, min ~180px)
- Collapsed: 40px icon strip; each icon button includes `aria-label` for accessibility
- Background: bgBase (#111927)
- Header: Efxmux logo with accent diamond, 28px height area
- Project items: 6px vertical gap, active item has green dot + accent left border (2px)
- Section labels: GeistMono 10px uppercase, letter-spacing 1.2px, textDim color
- Git change badges: M (statusYellow on statusYellowBg), U (textMuted on statusMutedBg), tinted 3px radius

### Main Panel
- Terminal tab bar: bgBase background, bottom-border active indicator (2px accent)
- Agent header card: bgElevated fill, 8px radius, 8px 12px padding, gradient icon (28x28, 6px radius), version + model text, status pill
- Terminal area: bgDeep (#0B1120), full-height minus tab bar and agent header
- Terminal containers div: absolute positioned, preserved for xterm.js mount

### Right Panel
- Tab bar: pill-style buttons, bgElevated fill for active, bgBorder border for active, textDim for inactive
- Content areas: bgDeep (#0B1120) background
- Split handle: horizontal, bgBorder color, draggable

### Diff Viewer
- Header: file path, status badge (Modified/Added/Deleted), +/- line count stats
- Hunk headers: diffHunkBg background, textDim color
- Addition lines: diffGreenBg background, 3px left border statusGreen, green line numbers at diffGreenLineno opacity
- Deletion lines: diffRedBg background, 3px left border diffRed, red line numbers at diffRedLineno opacity
- Context lines: no background, textMuted line numbers

### File Tree
- Header: folder path with textMuted, divider below
- Folder icons: inline SVG (not lucide-preact), textDim color
- File icons: inline SVG, textDim color, differentiated by extension (code vs text)
- Indentation: 20px per depth level
- Selected row: bgElevated background
- File size: textDim, right-aligned

### Add Project Modal
- Overlay: black at 50% opacity
- Card: bgElevated (#19243A) fill, 12px outer radius, shadow-2xl, max-width 520px
- Header: textPrimary title, divider below
- Input fields: bgDeep (#0B1120) fill, bgBorder border, 8px radius, 10px 12px padding
- Field labels: textMuted, 12px
- Footer: divider above, Cancel (ghost) + Add Project (accent fill) buttons
- Browse button: bgBorder background, textSecondary text

### Preferences Panel
- Same modal treatment as Add Project
- Section labels: GeistMono 10px uppercase
- Setting rows: flexbox with label left, control right
- Theme toggle: accent background when active
- Keycap badges: bgDeep fill, bgBorder border, 4px radius, GeistMono 10px
- Agent badge: gradient background (agentGradientStart to agentGradientEnd)

### Server Pane
- Strip (collapsed): single row, bgBase background, bgBorder top border, server name + status + toggle button
- Expanded: bgBase background, toolbar with action buttons, scrollable log area with bgDeep background
- Status indicators: green dot for running, red for stopped/crashed, yellow for starting

### Tab Bar (generic)
- Main panel variant: bottom-border active indicator (2px accent), bgBase background
- Right panel variant: pill-style, bgElevated fill + bgBorder border for active tab, transparent for inactive

## Token Architecture

Dual token system as specified in D-06 and D-07:

1. **`src/styles/app.css` @theme block** -- CSS custom properties consumed by Tailwind utility classes (`bg-bg`, `text-text-bright`, `border-border`, etc.)
2. **`src/tokens.ts`** -- TypeScript constants consumed by inline `style={{}}` props for hex+alpha opacity variants, gradients, and pixel-specific dimensions that do not map to Tailwind utilities

Rule: If a color has no alpha channel, use the Tailwind class via @theme. If a color includes alpha (e.g., `#3FB95020`), use `tokens.ts` via inline style.
