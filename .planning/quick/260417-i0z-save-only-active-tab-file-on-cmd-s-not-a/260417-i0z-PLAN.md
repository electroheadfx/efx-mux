# Quick Task 260417-i0z: Save only active tab on Cmd+S

## Root cause

`src/main.tsx:225` dispatches `editor-save` CustomEvent on document.
Every mounted `EditorTab` (all open editor tabs — they stay mounted with `display:none`) registers a `document.addEventListener('editor-save', …)` in `src/components/editor-tab.tsx:125-134`, which calls `triggerEditorSave(tabId)` for its own tabId.

Result: one Cmd+S fires save on every open editor tab. User sees "Saved CLAUDE.md / Saved README.md / Saved test.md" toasts.

## Fix

Replace event-broadcast with direct function call. `main.tsx` already knows the active tabId (`activeUnifiedTabId.value`, which is the full `editor-<timestamp>` id — same value registered in `saveCallbackMap`). Call `triggerEditorSave(activeUnifiedTabId.value)` directly; remove the listener in `editor-tab.tsx`.

Also need to focus the editor view before save (currently done in the listener) — do it via the exported `EditorView` from `editorViewMap` or keep focus implicit. Active tab editor should already be focused; skip focus call (keep fix minimal). If focus is needed, add a `focusEditor(tabId)` helper.
