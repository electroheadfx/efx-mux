## Spacing Scale

Standard scale (multiples of 4 from the set {4, 8, 16, 24, 32, 48, 64}):

| Token | Value | Usage |
|-------|-------|-------|
| md | 4px | Tight element spacing |
| xl | 8px | Default element spacing |
| 4xl | 16px | Container padding |

**Named Exceptions (justified deviations):**

| Token | Value | Justification |
|-------|-------|---------------|
| xs | 1px | Hairline gaps: used for 1px visual separators and hairline borders that would be invisible at 4px |
| sm | 2px | Minimal padding: used for 2px underline heights on active tabs and tight icon-to-text gaps where 4px feels too loose |
| lg | 6px | Compact section padding: existing project pattern from sidebar.tsx; 4px too tight, 8px too loose for dense file lists |
| 2xl | 10px | Gap between related items: existing project pattern; bridges the visual gap between 8px and 12px for mid-density layouts |
| 3xl | 12px | Compact section padding: sits between 8px and 16px for dense sidebar rows where 8px is too tight and 16px over-spaces; used for toast internal padding |
| 5xl | 20px | Modal padding: provides breathing room between 16px and 24px for medium-density modal content; existing project pattern |
| 6xl | 28px | Major section breaks: used for tab row minimum height (28px) and button heights; provides larger touch targets than 24px while being more compact than 32px |

**Phase 16-specific usage:**
- Tab row padding: 4px horizontal, 8px vertical
- Tab gap: 0px (tabs touch, separated by visual underline)
- Commit textarea padding: 8px
- Button padding: 8px vertical, 16px horizontal
- Section header padding: 4px vertical, 8px horizontal
- File row padding: 4px vertical, 4px horizontal
- Checkbox-to-label gap: 8px

## Component Inventory

### TabRow
- **Location:** Below EFXMUX header, above content
- **Tabs:** "Projects" | "Files" | "Git"
- **Active indicator:** 2px bottom border in accent color
- **Inactive text:** colors.textMuted (#8B949E)
- **Active text:** colors.textPrimary (#E6EDF3)
- **Hover state:** text brightens to textSecondary (#C9D1D9)
- **Hit target:** Full tab width, 28px height minimum

### StagedSection / ChangesSection
- **Collapsible:** Click header to toggle
- **Header format:** "STAGED (N)" or "CHANGES (N)" with count
- **Icon:** ChevronDown when open, ChevronRight when collapsed
- **Animation:** 150ms ease-out height transition

### GitFileCheckboxRow
- **Layout:** checkbox (16x16) + badge (18x18) + filename
- **Checkbox:** Native checkbox with accent-color CSS
- **Checked = staged:** In STAGED section
- **Unchecked = unstaged:** In CHANGES section
- **Click checkbox:** Calls stageFile() or unstageFile()
- **Badge colors:** Per status color table above

### CommitTextarea
- **Position:** Top of Git tab content, always visible
- **Height:** 64px default, resizable to 120px max
- **Placeholder:** "Commit message..."
- **Border:** 1px solid bgBorder (#243352)
- **Focus border:** 1px solid accent (#258AD1)
- **Font:** GeistMono 12px

### CommitButton
- **Label:** "Commit (N files)" where N = staged count
- **Disabled when:** N = 0 OR textarea empty
- **Enabled style:** accent background, white text
- **Disabled style:** bgSurface background, textDim text, cursor: not-allowed

### PushButton
- **Visibility:** Only when unpushed commits exist (check via git status)
- **Label:** "Push to origin"
- **Spinner:** 12px loader icon replaces text during operation
- **Position:** Below Commit button, 8px gap

### Toast
- **Position:** Bottom-right, 16px from edges
- **Width:** 280px
- **Background:** bgElevated with 1px bgBorder
- **Duration:** 4000ms auto-dismiss
- **Success icon:** CheckCircle in statusGreen
- **Error icon:** XCircle in diffRed
- **Error copy:** Includes recovery hint (e.g., "Run: ssh-add")

## Interaction States

### Tab Switching
1. Click inactive tab
2. Active tab state transfers (signal update)
3. Previous tab content unmounts, new tab content mounts
4. No animation (instant switch)

### Stage/Unstage Flow
1. User clicks checkbox on file row
2. Checkbox toggles optimistically
3. stageFile()/unstageFile() IPC call fires
4. On success: file moves between STAGED/CHANGES sections
5. On error: revert checkbox, show toast with error and recovery hint

### Commit Flow
1. User types in commit textarea
2. Commit button enables when: staged > 0 AND message.trim().length > 0
3. User clicks Commit button
4. Button shows loading state (spinner)
5. commit() IPC call fires
6. On success: clear textarea, refresh file lists, brief success indicator
7. On error: toast with error message

### Push Flow
1. Push button visible when unpushed commits exist
2. User clicks Push button
3. Button shows spinner, text hidden
4. push() IPC call fires
5. On success: toast "Pushed to origin/{branch}", refresh status
6. On AuthFailed: toast "Authentication failed. Run: ssh-add"
7. On PushRejected: toast "Push rejected. Pull first."
