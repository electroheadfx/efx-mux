# Phase 3 — UI Design Contract

> Visual and interaction contract for terminal theming. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla CSS custom properties) |
| Preset | not applicable |
| Component library | none (Arrow.js + vanilla DOM) |
| Icon library | none (no new icons in this phase) |
| Font | FiraCode Light 300, loaded via @font-face from `/fonts/FiraCode-Light.woff2` |

**Note:** This phase adds no new visual components. The "UI" is the theme system itself -- dynamic CSS custom properties, xterm.js theme application, and a dark/light chrome toggle via `data-theme` attribute.

## Theme.json Schema Contract

The canonical schema that all downstream consumers must implement:

```json
{
  "chrome": {
    "bg": "#282d3a",
    "bgRaised": "#363b3d",
    "border": "#3e454a",
    "text": "#8d999a",
    "textBright": "#92a0a0",
    "accent": "#258ad1",
    "font": "FiraCode Light",
    "fontSize": 14
  },
  "terminal": {
    "background": "#282d3a",
    "foreground": "#92a0a0",
    "cursor": "#258ad1",
    "selectionBackground": "#3e454a",
    "black": "#073642",
    "red": "#dc322f",
    "green": "#859900",
    "yellow": "#b58900",
    "blue": "#268bd2",
    "magenta": "#d33682",
    "cyan": "#2aa198",
    "white": "#eee8d5",
    "brightBlack": "#002b36",
    "brightRed": "#cb4b16",
    "brightGreen": "#586e75",
    "brightYellow": "#657b83",
    "brightBlue": "#839496",
    "brightMagenta": "#6c71c4",
    "brightCyan": "#93a1a1",
    "brightWhite": "#fdf6e3"
  }
}
```

**Source:** CONTEXT.md D-02, D-03.

**Validation rules:**
- Partial themes are valid. Missing keys fall back to the Solarized Dark defaults above.
- Invalid JSON keeps the current theme and logs a console warning (CONTEXT.md D-12).
- `chrome.font` and `chrome.fontSize` are optional; defaults are `"FiraCode Light"` and `14`.

## CSS Custom Property Mapping

Chrome properties set on `:root` via JavaScript at runtime:

| CSS Variable | theme.json Key | Default |
|-------------|----------------|---------|
| `--bg` | `chrome.bg` | `#282d3a` |
| `--bg-raised` | `chrome.bgRaised` | `#363b3d` |
| `--border` | `chrome.border` | `#3e454a` |
| `--text` | `chrome.text` | `#8d999a` |
| `--text-bright` | `chrome.textBright` | `#92a0a0` |
| `--accent` | `chrome.accent` | `#258ad1` |
| `--font` | `chrome.font` (wrapped: `'{value}', monospace`) | `'FiraCode', 'Fira Code', 'JetBrains Mono', monospace` |
| `--font-size` | `chrome.fontSize` (appended: `px`) | `14px` |

**Dark/light interaction:** When `data-theme="light"` is set, the CSS light-mode variable block provides the chrome colors. Theme.json chrome values override the dark mode defaults only. Light mode has its own static set defined in CSS.

## Dark/Light Toggle Interaction Contract

| Property | Value |
|----------|-------|
| Trigger | No visible toggle button in Phase 3. Toggle is callable via `toggleThemeMode()` JS function (exposed for Phase 8 UI). |
| Mechanism | `document.documentElement.setAttribute('data-theme', 'dark'|'light')` |
| Persistence | `localStorage.setItem('efxmux:theme-mode', 'dark'|'light')` |
| Restore | On startup, read `localStorage.getItem('efxmux:theme-mode')` before first paint. Default: `'dark'`. |
| Scope | Chrome only (sidebar, panel headers, borders, split handles). Terminal colors unaffected. |
| Transition | No CSS transition on color change. Instant swap. |

**Source:** CONTEXT.md D-13, D-14, D-15.

---

## Hot-Reload Interaction Contract

| Property | Value |
|----------|-------|
| Trigger | User saves `~/.config/efxmux/theme.json` in any editor |
| Debounce | 200ms (CONTEXT.md D-11) |
| Latency target | Theme applied within 1 second of file save (THEME-03 success criteria) |
| Watcher scope | `~/.config/efxmux/` directory, NonRecursive mode, filter for `theme.json` filename |
| Event name | `theme-changed` (Tauri event, payload is full theme JSON object) |
| Application order | (1) Update CSS custom properties on `:root`, (2) Set `terminal.options.theme` on all registered xterm.js instances, (3) Call `fitAddon.fit()` on all terminals if font size changed |
| Error behavior | Invalid JSON: log warning to console, keep current theme. No user-visible error UI. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable (no new UI controls in this phase) |
| Empty state heading | Not applicable (theme.json is auto-created with defaults on first launch) |
| Empty state body | Not applicable |
| Error state (invalid theme.json) | Console: `[efxmux] Invalid theme.json: {parse error}. Keeping current theme.` |
| Error state (file read failure) | Console: `[efxmux] Failed to read theme.json: {io error}. Keeping current theme.` |
| iTerm2 import success | Console: `[efxmux] Imported iTerm2 theme from {path}. Backup saved to theme.json.bak` |
| iTerm2 import failure | Console: `[efxmux] Failed to import iTerm2 theme: {reason}` |
| Destructive confirmation | Not applicable (no destructive actions; iTerm2 import creates `.bak` automatically per D-07) |

**Note:** All messaging is console-only in Phase 3. No toast/notification UI. Phase 8 may add user-visible error feedback.
